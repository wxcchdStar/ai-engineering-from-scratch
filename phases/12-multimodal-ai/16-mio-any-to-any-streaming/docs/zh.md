# MIO 与任意到任意流式多模态模型

> GPT-4o 提供了一个大多数开源模型无法复制的产品：一个实时听到语音、看到视频并进行回应的智能体。到 2024 年底，开源生态的答案是 MIO（Wang et al., 2024年9月）。MIO 将文本、图像、语音和音乐分别分词，在交织序列上训练一个因果 transformer，并实现任意模态到任意模态的生成。AnyGPT（Zhan et al., 2024年2月）是概念验证；MIO 是规模化版本；Unified-IO 2（Allen AI, 2023年12月）是带视觉 + 动作定位的近亲。本课解读任意到任意模式——四个分词器、一个 transformer、流式友好的解码。

**类型：** 学习
**语言：** Python（标准库，四模态 token 分配器 + 流式解码循环）
**前置课程：** Phase 12 · 11（Chameleon），Phase 6（语音与音频）
**时间：** ~120 分钟

## 学习目标

- 设计一个承载文本、图像、语音和音乐 token 且无碰撞的共享词表。
- 比较 SEED-Tokenizer（图像）和 SpeechTokenizer 残差 VQ（语音）在压缩与重建上的权衡。
- 解释构建任意到任意生成能力的四阶段课程。
- 列出三个开源任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题所在

统一多模态模型容易宣称但难以大规模构建。2024 年之前大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每次转换都丢失信息、增加延迟、使训练复杂化。GPT-4o 的演示视频展示了一个亚秒响应的单模型替代方案；开源系统落后了数月。

工程挑战：

- 必须为每种模态准备分词器，压缩足够无损以支持重建，且产出 token 的速率是 transformer 能消费的。
- 单一词表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。最少四万多个条目。
- 训练数据必须覆盖每种输入-输出对（文本→图像、图像→语音、语音→图像等），否则模型必须能组合泛化。
- 推理必须足够快地流式输出 token 以满足对话延迟（首个音频字节 <500ms）。

## 核心概念

### 四种模态的四个分词器

MIO 的分词器栈：

- 文本：标准 BPE，词表约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 个条目，每张图 32×32 个 token。
- 语音：SpeechTokenizer 残差 VQ（2023）——将 16kHz 波形编码为 8 个层次化码本；第一层是粗粒度内容，后续层添加韵律和说话人身份。
- 音乐：类似的残差 VQ（Meta 的 MusicGen / Encodec 家族），4-8 个码本。

每种模态产出整数 token。这些 token 在共享词表中获得不相交的 ID 范围：

```
text:   0..31999
image:  32000..36095  (4096 个图像 token)
speech: 36096..40191  (4096 个语音基础 token，加残差层)
music:  40192..48383  (8192 个音乐 token)
sep:    48384..48390  (<image>, <speech>, <music>, </...> 等)
```

总计：约 48k 词表。输入嵌入和输出投影覆盖全部。

### 流式解码

语音生成使用残差 VQ。Transformer 预测基础（第 0 层）语音 token；并行解码的残差量化器预测后续层。每个第 0 层 token 在 16kHz 下大约对应 50ms 音频。

流式模式：

1. 用户对麦克风说话；实时音频分词器每 50ms 输出语音 token。
2. MIO 在 token 到达时消费它们（提示预填充 + 增量前向传播）。
3. 输出 token 生成后立即流出；并行语音解码器以约 50-150ms 延迟将它们转换为音频采样。
4. 首个音频字节时间 (TTFAB)：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音 LLM 设计。Moshi 特别在单 GPU 上实现了 160ms 往返延迟。

### 四阶段课程

MIO 的训练课程：

1. 阶段 1 —— 对齐。大规模模态对语料：文本-图像、文本-语音、文本-音乐。每对使用各自的 token 词表段。训练共享词表。
2. 阶段 2 —— 交织。多模态交织文档（带图像+视频的博客、带转录的播客等）。训练跨模态上下文。
3. 阶段 3 —— 语音增强。额外音频数据以在不丧失文本能力的前提下提升语音质量。
4. 阶段 4 —— SFT。跨模态指令微调：VQA、描述、叙述、语音到语音对话。

跳过一个阶段会降低特定能力：跳过阶段 2 模型丧失跨模态上下文；跳过阶段 3 语音会很差。

### 视觉思维链

MIO 引入了视觉思维链 (chain-of-visual-thought)：模型作为推理步骤输出中间图像 token。对于"猫在爬树吗？"这个问题，模型：

1. 输出渲染场景的 `<image>` token（来自输入图像或草图）。
2. 输出分析草图的文本。
3. 输出最终答案。

渲染的中间图像充当草稿本。空间推理任务的基准测试得到改善。这个想法与文本推理中的思维链对应。

### 任意到任意领域的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），设计类似。
- Unified-IO 2（arXiv:2312.17172）：添加了视觉动作输出、深度、法线。更多任务多样性，较小规模。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态特定扩散解码器。非单模型方案。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜在空间实现任意到任意。

MIO 是最接近纯 token 任意到任意的。AnyGPT 是其概念祖先。

### 延迟预算

对于对话产品，每个组件的延迟都很重要：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史）：8B 模型上约 100ms。
- 首个输出 token：约 50ms。
- 并行残差 VQ + 语音解码器：约 100-150ms。

首个音频字节总延迟：最少约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 在公开基准中约 400-600ms。

### 为什么任意到任意仍然困难

即使在 2026 年，开源任意到任意模型在两个维度上落后于闭源模型：

- 语音质量。残差 VQ 分词器是有损的；对话语音听起来比 ElevenLabs 级别的声音机械。
- 跨模态推理。要求模型"唱出你看到的"仍然比纯视觉任务更容易失败。

这些是开放的研究问题。Qwen3-Omni（Lesson 12.20）是 2025 年最先进的开源尝试。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是什么？列出花费时间的组件。

2. SpeechTokenizer 残差 VQ 使用 8 个码本。解释为什么并行解码残差层是必要的（vs 顺序）以及它带来多少延迟节省。

3. 你的词表有 32k 文本 + 4k 图像 + 4k 语音。添加 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数成本是多少？

4. 视觉思维链输出中间图像。哪类问题受益？哪类问题被额外 token 伤害？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技术并与 MIO 的视觉思维链比较。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|----------|----------|
| 任意到任意 (Any-to-any) | "多模态输入/输出" | 一个模型在任意方向上接受和输出文本、图像、语音和音乐 |
| 残差 VQ (Residual-VQ) | "语音分词器栈" | 多码本分词，每层添加信息；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像码" | MIO 使用的 4096 条目码本离散图像分词器 |
| 视觉思维链 (Chain-of-visual-thought) | "视觉草稿本" | 模型在最终答案前生成中间图像作为推理步骤 |
| 首个音频字节时间 (Time-to-first-audio-byte) | "TTFAB" | 从用户说话到第一个音频输出的延迟；<500ms 才有对话感 |
| 四阶段课程 (Four-stage curriculum) | "训练方案" | 对齐 → 交织 → 语音增强 → SFT，按此顺序 |
