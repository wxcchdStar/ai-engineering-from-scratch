# InternVL3：原生多模态预训练

> 在 InternVL3 之前，每个开源 VLM 都遵循相同的三步方案：拿一个在数万亿文本 token 上训练好的文本 LLM，装上视觉编码器，然后微调接缝处。这能用，但存在对齐债务 (alignment debt)——文本 LLM 将其全部预训练预算花在了纯文本上，并不原生理解视觉 token。当你事后添加视觉时，LLM 必须在不遗忘文本能力的前提下重新学习如何将视觉输入与其文本推理关联。InternVL3（Zhu et al., 2025年4月）拒绝了事后方案：一次预训练，从第一步起文本和多模态交织。结果在 78B 参数开源模型上匹配 Gemini 2.5 Pro 的 MMMU-Pro 分数。本课解读原生预训练的论据以及采用它之后发生了什么变化。

**类型：** 学习
**语言：** Python（标准库，训练语料混合器）
**前置课程：** Phase 12 · 05，Phase 12 · 07（方案汇总）
**时间：** ~120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会积累对齐债务，引用三个可量化的症状（灾难性遗忘、回答漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料混合比以及文本:交织:描述比例为什么重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说明视觉分辨率路由器 (ViR) 和解耦视觉-语言 (DvD) 部署优化。

## 问题所在

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——全都拿一个已预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选的任务特定微调。

对齐债务的三个症状会显现：

- 灾难性遗忘 (Catastrophic forgetting)。事后 VLM 遗忘纯文本技能。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本智能体退化。
- 回答漂移 (Answer drift)。同一视觉问题的细微措辞变化得到不同答案。视觉编码器与 LLM 的连接绑定比 LLM 自身 token 之间的绑定更弱。
- 视觉-文本不一致 (Visual-text inconsistency)。VLM 可以正确描述一张图像，然后在回答问题时与自己的描述矛盾。视觉 token 不像文本那样参与 LLM 的内部一致性检查。

这些症状已有充分记录。MM1.5 第 4 节量化了它们。LLaVA-OneVision 的消融实验暗示了它们。原生预训练是答案。

## 核心概念

### 原生多模态预训练

InternVL3 从头开始训练，语料从第一步起就是原生多模态的。混合比为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交织图文数据（OBELICS、MMC4 风格）
- 20% 配对图像-描述数据
- 5% 视频-文本数据

视觉 token、文本 token 和跨模态交互从第一个梯度步起就参与相同的损失函数。无需对齐预训练、无需投影器冻结阶段、无需从灾难性遗忘中恢复。

基座模型的训练是单阶段的。指令微调在后面，但基座模型已经将视觉 token 视为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用固定轴分配的 M-RoPE。InternVL3 引入 V2PE：位置编码按模态类型（文本、图像、视频）变化，带可学习的缩放。实践中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享相同的 RoPE 频率基数，但每个频带的隐藏维度分配是一个可学习参数而非固定分割。在预训练期间可以自由地权衡时间 vs 空间频率分辨率。

V2PE 的消融结果：在相同计算量下，视频基准上比 M-RoPE 提升 1-2 分。不是革命，但更干净。

### 视觉分辨率路由器 (ViR)

部署优化。并非所有图像都需要全分辨率编码。一张只有一个简单物体的低细节照片在 1280px 原生编码时浪费 token。ViR 是一个小型分类器，在编码之前预测回答问题所需的最低分辨率。

路由有三个层级：低分辨率（256 token）、中等（576）、高（2048+）。在生产流量中 60% 的查询用低或中等就足够。净效果：在相同质量下吞吐量提升 2-3 倍。

### 解耦视觉-语言部署 (DvD)

当你服务一个大型 VLM 时，视觉编码器每张图像运行一次，但 LLM 对每个输出 token 都自回归运行。两个组件有不同的瓶颈（视觉 = GPU 显存带宽用于 conv + attention；LLM = KV cache）。DvD 将它们分到不同的 GPU 上，之间用流式传输。

对于一个 8B + 400M 编码器的模型，DvD 大约使单节点吞吐量翻倍（vs 共置部署）。

### 单阶段 vs 多阶段的质量

InternVL3 的主要基准声明：在 78B 参数下匹配 Gemini 2.5 Pro 的 MMMU-Pro。在 38B 匹配 GPT-4o。在 8B 领先开源 8B 排行榜。全部基于单阶段预训练 + 指令微调方案。

对齐债务假说是可量化的：InternVL3-8B 每单位视觉基准提升所损失的文本基准分（MMLU、GSM8K）比 Qwen2.5-VL-7B 少。该模型更通用，因为训练是一体的，而非两段式。

### InternVL3.5 和 InternVL-U

InternVL3.5（2025年8月）缩放了该方案。相同的原生预训练方法，更多数据，更多参数。MMMU 的提升是增量的。

InternVL-U（2026）添加了统一生成——在相同骨干上通过 MMDiT 头实现图像输出。"U"代表"理解 + 生成 (Understanding + generation)"，追赶 Transfusion 风格的统一模型（Lesson 12.13）。同一个原生预训练骨干同时支持理解和生成头。

### 原生预训练的权衡

原生预训练不是免费的：

- 计算量。从头训练一个新 VLM 的成本等同于训练一个文本 LLM——数百万 GPU 小时。事后适配复用现有 LLM 权重，节省大部分成本。
- 数据。大规模的交织图文语料稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。纯文本有 15T token 级别。多模态预训练数据稀缺是硬约束。
- 基座 LLM 复用。原生预训练放弃了日后换入新 LLM 的选项。事后方案让你可以通过只重训适配器就将 Llama-3.1 换成 Llama-4。

InternVL3 的赌注是：对齐债务比复用损失更严重。基准测试支持这一说法。高昂的训练成本阻止了其他实验室廉价复制。事后 VLM 会继续存在，因为对大多数项目来说它们仍然更便宜。

## 上手实践

`code/main.py` 是一个训练语料混合器和 ViR 路由模拟器。它能：

- 接受目标语料混合比（%文本、%交织、%描述、%视频）并计算每种模态的预期步数。
- 在一批查询上模拟 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节）并报告平均 token 数。
- 给定编码器 vs LLM 的 FLOPs，报告 DvD 吞吐量估算。
- 并排打印事后方案 vs 原生预训练在参数、计算量、数据和预期对齐债务症状上的对比。

## 交付产出

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定一个拟议的 VLM 训练计划，它审计应该走原生还是事后路线，标记对齐债务风险，并推荐语料混合比。在规模化一个新的开源 VLM 项目并需要选择训练策略时使用。

## 练习

1. 估算 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后训练）之间的计算量差异。GPU 小时的大致比率？什么解释了这个差距？

2. InternVL3 报告 40% 文本 / 35% 交织 / 20% 描述 / 5% 视频。如果你的目标任务是视频为主的，提出新的比例并论证为什么基座模型仍然需要大量文本和描述数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。指出事后训练显示最大退化的确切基准测试。退化造成了多少损失？

4. ViR 将 60% 的流量路由到低分辨率编码。哪类查询会被误路由（应该用高分辨率却被送去低分辨率）？提出三种路由器失败模式。

5. DvD 将视觉和 LLM 分到不同 GPU。在什么流量模式下 DvD 会降低而非提升吞吐量？

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|----------|----------|
| 原生多模态预训练 (Native multimodal pretraining) | "从头一起训" | 文本 + 图像 + 视频 token 从第 1 步起就参与损失函数，而非事后挂接 |
| 对齐债务 (Alignment debt) | "事后惩罚" | 将视觉挂接到冻结 LLM 上带来的可量化退化：文本技能和回答一致性的下降 |
| V2PE | "可变视觉位置编码" | 按模态可学习的位置编码分配；InternVL3 对 M-RoPE 的继任者 |
| ViR | "分辨率路由器" | 在编码之前预测每个查询所需最低分辨率的小型分类器，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一块 GPU，LLM 在另一块，之间流式交接；大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年后续工作，在原生预训练骨干上添加图像生成头 |
| 交织语料 (Interleaved corpus) | "OBELICS / MMC4" | 文本和图像按自然阅读顺序排列的文档；原生预训练的原材料 |
