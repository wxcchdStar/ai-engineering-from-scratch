# 从 CLIP 到 BLIP-2 —— Q-Former（查询Transformer） 作为模态桥接

> CLIP 能对齐图像和文本，但无法生成字幕、回答问题或进行对话。BLIP-2（Salesforce，2023）用一个小的可训练桥接解决了这个问题：32 个可学习查询向量通过交叉注意力关注冻结 ViT 的特征，然后直接插入冻结 LLM 的输入流中。1.88 亿参数的桥接连接了一个 110 亿参数的 LLM 和一个 ViT-g/14。直到 2026 年，每一个基于适配器的 VLM——MiniGPT-4、InstructBLIP、LLaVA 的各种变体——都是它的后代。本课将解读 Q-Former（查询Transformer） 的架构，解释其两阶段训练过程，并构建一个将视觉 token 输入冻结文本解码器的玩具版本。

**类型：** 构建
**语言：** Python（标准库，交叉注意力 + 可学习查询 演示）
**前置课程：** Phase 12 · 02（CLIP），Phase 7（Transformers）
**时间：** 约 180 分钟

## 学习目标

- 解释为什么在冻结视觉编码器和冻结 LLM 之间设置一个可训练瓶颈，在成本和稳定性上优于端到端微调。
- 实现一个交叉注意力块，其中一组固定的可学习查询关注外部图像特征。
- 走读 BLIP-2 的两阶段预训练：表示学习（ITC（图文对比） + ITM（图文匹配） + ITG（图像条件文本生成）），然后是生成式学习（冻结解码器上的 LM 损失）。
- 对比 Q-Former（查询Transformer） 与 LLaVA 中使用的更简单的 MLP 投影器，并论证每种选择在什么情况下胜出。

## 问题所在

你有一个冻结的 ViT，每张图像输出 256 个 patch token，每个维度为 1408。你有一个冻结的 70 亿参数 LLM，它期望维度为 4096 的 token 嵌入。最直接的桥接——一个从 1408 到 4096 的线性层——是可行的，但将所有 256 个 patch token 送入 LLM 的上下文，每张图像要额外消耗 256 个 token。在 32 张图像的批次中，仅视觉模态就消耗了 8192 个 token。

BLIP-2 的核心问题：能否将 256 个 token 的图像表示压缩成少得多的 token（比如 32 个），同时保留足够的信息，让 LLM 能够为图像生成字幕、回答问题并进行推理？而且能否在不触碰冻结骨干的情况下训练这个桥接，使训练成本仅局限于桥接的参数？

答案就是：Q-Former（查询Transformer）。32 个可学习的"查询"向量通过交叉注意力关注 ViT 的 patch token，生成一个 32 个 token 的视觉摘要供 LLM 消费。总参数量 1.88 亿。在接触 LLM 之前，先用对比、匹配和生成目标进行训练。

## 核心概念

### 可学习查询

Q-Former（查询Transformer） 的核心技巧：不是让 LLM 的文本 token 去关注图像 patch，而是引入一组新的 32 个可学习查询向量 `Q`，让它们去关注图像 patch。这些查询是模型的参数——它们在训练过程中被学习，每张图像都使用相同的 32 个查询。

经过交叉注意力后，每个查询持有一份图像的压缩摘要——"描述主要物体"、"描述背景"、"数物体数量"等。这些查询并不会字面意义上专门对应语义标签；它们学到的是任何能使下游损失下降的编码方式。

### 架构

Q-Former（查询Transformer） 是一个小型 transformer（12 层，约 1 亿参数），有两条路径：

1. 查询路径：32 个查询向量经过自注意力（在它们之间），然后通过交叉注意力关注冻结 ViT 的 patch token，最后经过 FFN。
2. 文本路径：一个类似 BERT 的文本编码器与查询路径共享自注意力和 FFN 权重。文本路径不使用交叉注意力。

训练时两条路径同时运行。查询和文本通过共享的自注意力进行交互，这意味着查询可以根据文本进行条件化，以完成需要文本的任务（ITM（图文匹配）、ITG（图像条件文本生成））。在推理时用于 VLM 交接的场景中，只有查询路径流动，产生 32 个视觉 token。

### 两阶段训练

BLIP-2 分两个阶段进行预训练：

第一阶段：表示学习（不涉及 LLM）。三种损失函数：
- ITC（图文对比）：CLIP 风格的对比，在池化后的查询 token 和文本 CLS token 之间进行。
- ITM（图文匹配）：二分类器——这对图文是否匹配？使用难负样本挖掘。
- ITG（图像条件文本生成）：在文本上使用因果 LM 头，以查询为条件。迫使查询编码可被文本生成的内容。

只有 Q-Former（查询Transformer） 参与训练。ViT 被冻结。不涉及 LLM。

第二阶段：生成式学习。接入一个冻结的 LLM（OPT-2.7B 或 Flan-T5-XL 等）。通过一个小型线性层将 32 个查询输出投影到 LLM 的嵌入维度。将它们前置到文本提示之前。仅在拼接后的提示 + 图像 + 字幕序列上，对线性投影和 Q-Former（查询Transformer） 进行 LM 损失训练。

第二阶段之后，Q-Former（查询Transformer） + 投影层就是完整的视觉适配器。推理流程：图像 → ViT → Q-Former（查询Transformer） → 线性投影 → 前置到文本之前 → 冻结 LLM 输出结果。

### 参数量经济

BLIP-2 使用 ViT-g/14（11 亿参数，冻结） + OPT-6.7B（67 亿参数，冻结） + Q-Former（查询Transformer）（1.88 亿参数，训练） = 总共 80 亿参数，其中 1.88 亿被训练。Q-Former（查询Transformer） 单独仅占整个技术栈参数的约 2.4%。训练成本反映了这一点：在几块 A100 上训练几天，对比端到端训练需要数周。

质量：BLIP-2 在零样本 VQA 上达到或超越了 Flamingo-80B，而体积只有后者的 1/50。这个桥接是有效的。

### InstructBLIP 与指令感知的 Q-Former（查询Transformer）

InstructBLIP（2023）为 Q-Former（查询Transformer） 增加了一个额外输入：指令文本本身。在交叉注意力阶段，查询现在可以同时访问图像 patch 和指令。查询可以按指令进行专门化（"数车"、"描述氛围"），而不是学习一个固定的摘要。在留出任务上取得了基准测试的提升。

### MiniGPT-4 与纯投影器方法

MiniGPT-4 保留了 Q-Former（查询Transformer），但只训练输出线性投影层，冻结其他所有部分。成本低廉，但代价是质量——查询是 BLIP-2 的，不是你自己的。适合快速迭代，但不是最佳架构。

### LLaVA 为什么选择了更简单的方式

LLaVA（2023，第 12.05 课）用一个普通的 2 层 MLP 投影器 替代了 Q-Former（查询Transformer），将每个 ViT patch token 投影到 LLM 空间——对于 24×24 的网格，每张图像 576 个 token，全部送入 LLM。压缩效果更差，但让 LLM 可以关注原始 patch。这在当时是有争议的；到 2023 年底，它成了主流，因为视觉指令数据（LLaVA-Instruct-150k）证明了 MLP 投影器 可以被训练以保留足够的信号。权衡在于：LLaVA 的上下文消耗更快，但它能自然地扩展到多图像和视频场景。

到 2026 年，领域出现分化：Q-Former（查询Transformer） 在 token 预算紧张的场景中存活（长视频、多图像）；MLP 投影器 在优先考虑每 token 原始质量的场景中占据主导。

### 门控交叉注意力：Flamingo，先驱者

Flamingo（第 12.04 课）早于 BLIP-2，使用了相同的交叉注意力思想，但将其放在每一层冻结 LLM 中，而不是作为单一的桥接。BLIP-2 证明了你可以只压缩到输入层，仍然有效。Gemini 和 Idefics 结合了两者：交错的输入 token 加上可选的用于上下文少样本学习的门控交叉注意力。

### 2026 年的后代家族

- Q-Former（查询Transformer）：BLIP-2、InstructBLIP、MiniGPT-4，以及大多数因 token 预算原因而采用的视频语言模型。
- Perceiver 重采样器：Flamingo 的变体（第 12.04 课）；Idefics 家族、Eagle、OmniMAE。
- MLP 投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

四种方案都是有效的。决定性的问题是：你的约束是 token 预算还是每 token 质量。

## 使用方式

`code/main.py` 构建了一个基于标准库的 Q-Former 风格交叉注意力：

1. 模拟 256 个图像 patch token（维度 128）。
2. 实例化 32 个可学习查询向量（维度 128）。
3. 执行缩放点积交叉注意力（Q 来自查询向量，K/V 来自 patch）。
4. 通过线性层投影到 LLM 维度（512）。
5. 输出 32 个可供 LLM 使用的视觉 token。

所有数学运算均使用纯 Python 实现（向量嵌套循环）。代码虽精简，但形状正确。注意力权重矩阵会被打印出来，让你能看到每个查询从哪些 patch 中提取了信息。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| Q-Former | "查询 Transformer" | 一个小型 Transformer，包含 32 个可学习查询向量，对冻结的 ViT 特征进行交叉注意力 |
| 可学习查询 | "视觉的软提示" | 一组固定参数，作为交叉注意力的查询端；按模型学习，所有输入共享 |
| 交叉注意力 | "Q 来自这里，K/V 来自那里" | 查询、键、值来自不同来源的注意力机制；查询向量如何从 ViT patch 中提取信息 |
| ITC | "图像-文本对比" | 应用于 Q-Former 池化查询与文本 CLS 的 CLIP 风格损失 |
| ITM | "图像-文本匹配" | 在难负例挖掘得到的样本对上的二分类器；迫使查询向量区分细粒度不匹配 |
| ITG | "以图像为条件的文本生成" | 因果语言模型损失，文本生成以查询向量为条件；迫使查询向量编码可被文本解码的内容 |
| 两阶段预训练 | "先表示学习，再生成学习" | 第一阶段单独训练 Q-Former（ITC/ITM/ITG）；第二阶段接入冻结的 LLM，仅训练投影层 + Q-Former |
| 冻结骨干网络 | "不微调" | 视觉编码器和 LLM 的权重固定不变；仅训练桥接模块 |
| 投影头 | "线性映射到 LLM 维度" | 将 Q-Former 输出映射到 LLM 嵌入维度的最终线性层 |
| Perceiver resampler | "Flamingo 的版本" | 类似的可学习查询交叉注意力，由 Flamingo 在每一层中使用，而非作为单一桥接模块 |
