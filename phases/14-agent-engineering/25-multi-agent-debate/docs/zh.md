# 多 Agent 辩论与协作 (Multi-Agent Debate and Collaboration)

> Du et al. (ICML 2024, "Society of Minds") 运行 N 个模型实例独立提出答案，然后在 R 轮中迭代相互批评以收敛。提高事实性、规则遵循、推理能力。稀疏拓扑在 token 成本上优于全网格。

**类型：** 学习 + 构建 (Learn + Build)
**语言：** Python (标准库)
**前置课程：** Phase 14 · 12 (工作流模式), Phase 14 · 05 (Self-Refine 与 CRITIC)
**时间：** 约 60 分钟

## 学习目标

- 解释辩论协议：N 个提议者，R 轮，收敛到共享答案。
- 描述为什么辩论能提高事实性、规则遵循和推理能力。
- 解释稀疏拓扑：并非每个辩论者都需要看到所有其他辩论者。
- 用标准库在脚本化 LLM 上实现全网格和稀疏变体的辩论；测量 token 成本 vs 准确率。

## 问题

Self-Refine (第 05 课) 是一个模型批评自己——有群体思维的风险。CRITIC (第 05 课) 将批评锚定在外部工具上——并非总是可用。辩论引入了第三种模式：多个实例，交叉批评，通过分歧收敛。

## 概念

### Society of Minds (Du et al., ICML 2024)

- N 个模型实例独立对同一问题提出答案。
- 在 R 轮中，每个模型阅读其他模型的提案并批评它们。
- 模型基于批评更新其答案。
- R 轮后，返回收敛的答案。

原始实验因成本使用 N=3, R=2。在困难问题上（MMLU、GSM8K、国际象棋走法有效性、传记生成），准确率随更多 Agent 和更多轮次而提高。

跨模型组合优于单模型辩论：ChatGPT + Bard 一起 > 单独任何一个。

### 稀疏拓扑

"Improving Multi-Agent Debate with Sparse Communication Topology" (arXiv:2406.11776, 2024-2025) 表明全网格辩论并非总是最优。稀疏拓扑（星形、环形、中心辐射型）可以在更低的 token 成本下匹配准确率。每个辩论者只看到一部分同伴。

影响：

- 全网格 N=5, R=3 = 5 × 3 = 15 个提案，每个阅读 4 个同伴 = 60 次批评操作。
- 星形 N=5, R=3（一个中心 + 4 个辐射）= 15 个提案，辐射只阅读中心 = 12 次批评操作。

### 辩论有帮助的场景

- **事实性。** N 个独立提案，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走法有效性——一个模型漏掉规则，其他模型捕获。
- **开放式推理。** 多种框架缩小到正确答案。

### 辩论有害的场景

- **延迟敏感的 UX。** N × R 串行轮次是你可能没有的延迟。
- **成本敏感的规模。** 每个问题 N × R 个 token。
- **简单事实查找。** 一次查找比五次辩论便宜。

### 2026 年实践实例

- **Anthropic 编排器-工作者** (第 12 课) — 带综合步骤的辩论变体。
- **LangGraph 监督者** (第 13 课) — 中心路由器 + 专家 Agent 可以将辩论实现为一个节点。
- **OpenAI Agents SDK** (第 16 课) — Agent 来回交接进行迭代批评。
- **多 Agent 评估** — 将辩论 + 评估器-优化器配对用于评估信号。

### 这个模式哪里会出错

- **收敛崩溃。** 所有 Agent 收敛到第一个错误答案。通过要求分歧轮次来缓解。
- **中心故障。** 在星形拓扑中，一个坏的中心污染所有人。轮换或使用多个中心。
- **Prompt 同质化。** 所有 Agent 使用相同的 prompt；它们产生相同的答案。使用多样化的 prompt 和/或模型。

## 构建

`code/main.py` 用标准库实现辩论：

- `Debater` 类（脚本化 LLM，每个辩论者有意见漂移）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实性的、一个基于规则的、一个推理的。
- 指标：收敛答案、收敛轮次、总批评操作数。

运行：

```
python3 code/main.py
```

输出：每个协议的准确率和成本；稀疏在 2/3 问题上以更低成本匹配全网格。

## 使用

- **Anthropic 编排器-工作者** 用于简单的 2-3 工作者辩论。
- **LangGraph** 用于带检查点的有状态多轮辩论。
- **自定义** 用于研究或专门正确性保证。

## 交付

`outputs/skill-debate.md` 搭建一个多 Agent 辩论，包含可配置拓扑、N、R 和收敛规则。

## 练习

1. 实现"强制分歧"规则：在第 1 轮，每个辩论者必须产生不同的提案。测量对收敛速度的影响。
2. 添加置信度加权聚合：辩论者返回 (答案, 置信度)；聚合器按置信度加权。有帮助吗？
3. 将一个"Agent"替换为具有不同意见的不同脚本化 LLM。异质性是否提高准确率？
4. 在你的 3 个问题上测量全网格 vs 稀疏的 token 成本。绘制成本 vs 准确率图。
5. 阅读 Society of Minds 论文。将你的玩具移植到 N=5, R=3。什么坏了？什么变好了？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 辩论 (Debate) | "多 Agent 批评" | N 个提议者，R 轮交叉批评，收敛 |
| 全网格 (Full mesh) | "每个人读每个人" | 每个辩论者每轮阅读每个同伴 |
| 稀疏拓扑 (Sparse topology) | "有限的同伴视图" | 辩论者只阅读一部分同伴 |
| 中心辐射型 (Hub-and-spoke) | "星形拓扑" | 一个中心辩论者，N-1 个辐射只阅读中心 |
| 收敛 (Convergence) | "一致" | 辩论者收敛到共享答案 |
| Society of Minds | "Du et al. 辩论论文" | ICML 2024 多 Agent 辩论方法 |
