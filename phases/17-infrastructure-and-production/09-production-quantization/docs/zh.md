# 生产量化 — INT8、INT4、GPTQ、AWQ、SmoothQuant 与 GGUF

> 量化（Quantization）通过降低数值精度来压缩 LLM，以换取在可接受精度损失范围内的更快推理。2026 年的生产格局：**INT8 量化** 是安全默认值 — 精度损失 < 0.5%，吞吐量提升 1.5-2 倍。**INT4 量化** 是激进选择 — 精度损失 1-5%，吞吐量提升 2-4 倍。**GPTQ（GPT 后训练量化，GPT Post-Training Quantization）** 是一种基于 Hessian 矩阵的权重量化方法，逐层优化以最小化输出误差 — 需要校准数据，但产生高质量的 INT4 模型。**AWQ（激活感知权重量化，Activation-aware Weight Quantization）** 观察到 1% 的显著权重（salient weights）贡献了大部分输出质量 — 它保护这些权重（保持更高精度），同时积极量化其余权重。**SmoothQuant** 通过将激活异常值平滑到权重中来处理激活异常值问题，实现 W8A8（INT8 权重 + INT8 激活）量化。**GGUF（GPT 生成统一格式，GPT-Generated Unified Format）** 是 llama.cpp 的文件格式 — 它将量化权重与模型架构捆绑在一个文件中，支持 K-quant（K 均值量化）变体（Q4_K_M、Q5_K_M 等）。精度与速度的权衡取决于任务 — 事实性问答对量化具有惊人的鲁棒性（INT4 精度损失 < 1%）；数学推理退化更严重（INT4 精度损失 3-8%）。

## 学习目标

- 区分 INT8（安全默认值）和 INT4（激进选择），并量化每种方案的精度损失和吞吐量提升。
- 解释 GPTQ（基于 Hessian 矩阵）和 AWQ（保护显著权重）之间的方法差异。
- 描述 SmoothQuant 如何通过将激活异常值平滑到权重中来处理激活异常值。
- 说出 GGUF 格式及其 K-quant 变体在 llama.cpp 生态系统中的作用。

## 问题

Llama 3.3 70B FP16 占用约 140 GB。你需要 2 个 A100（80 GB）才能加载它。INT8 量化将其减少到约 70 GB — 1 个 A100。INT4 量化将其减少到约 35 GB — 1 个 A100，还有空间容纳大批量或长上下文。

但量化不是免费的。INT4 在 MMLU 上损失 1-3%，在 GSM8K（数学推理）上损失 3-8%。对于客户支持聊天机器人来说，这是可以接受的。对于数学辅导应用来说，这是不可接受的。

诀窍在于理解哪种量化方法适合你的工作负载，以及精度损失在哪里是可以接受的。

## 概念

### 量化精度级别

**FP16**：基线。16 位浮点数。140 GB 用于 70B 模型。无精度损失。

**INT8**：8 位整数。70 GB 用于 70B 模型。精度损失 < 0.5%。吞吐量提升 1.5-2 倍。安全默认值 — 如果你不确定，使用 INT8。

**INT4**：4 位整数。35 GB 用于 70B 模型。精度损失 1-5%（任务相关）。吞吐量提升 2-4 倍。激进选择 — 在部署前进行基准测试。

**FP4**：4 位浮点数（Blackwell 原生，第 17 阶段 · 07）。35 GB。精度损失与 INT4 相似，但 Microscaling 格式提供更好的动态范围。

### GPTQ — 基于 Hessian 矩阵的量化

GPTQ 使用 Hessian 矩阵（二阶导数）信息来最小化量化误差。流程：
1. 通过模型运行校准数据（100-1000 个样本）。
2. 对于每一层，计算 Hessian 矩阵以确定哪些权重对输出最敏感。
3. 逐层量化权重，补偿之前量化的权重引入的误差。

结果：高质量的 INT4 模型，困惑度退化最小。缺点：需要校准数据；量化过程较慢（70B 模型需要数小时）。

### AWQ — 激活感知权重量化

AWQ 观察到权重的重要性不均匀 — 约 1% 的权重（对应大激活值的通道）贡献了大部分输出质量。AWQ：
1. 通过校准数据运行模型，收集激活统计信息。
2. 识别显著权重（对应大激活值的通道）。
3. 在量化前按比例放大显著权重（保护它们），按比例缩小其他权重。
4. 应用均匀量化。

结果：比 GPTQ 更简单（无需 Hessian 矩阵），质量相似。已成为 vLLM 和 SGLang 的默认量化方法。

### SmoothQuant — 处理激活异常值

LLM 激活具有异常值 — 某些通道的值比其他通道大 100 倍。量化激活（W8A8）会因为这些异常值而失败。SmoothQuant：
1. 识别激活异常值通道。
2. 通过数学等效变换将量化难度从激活平滑到权重中：将激活缩小 s 倍，将权重放大 s 倍。
3. 结果：激活和权重都可以用 INT8 表示。

实现 W8A8（INT8 权重 + INT8 激活），吞吐量比 W8A16（仅 INT8 权重）更高。

### GGUF — llama.cpp 格式

GGUF 是 llama.cpp 的文件格式。它将量化权重 + 模型架构 + 分词器 + 元数据捆绑在一个文件中。K-quant 变体使用 K 均值聚类进行非均匀量化：
- Q4_K_M：4 位，中等大小。良好的质量/大小权衡。
- Q5_K_M：5 位，中等大小。比 Q4 更好的质量。
- Q8_0：8 位。接近无损。

GGUF 模型在 llama.cpp 和 Ollama 中运行。不适合 vLLM（vLLM 使用 AWQ/GPTQ 格式）。

### 任务相关的精度损失

**事实性问答**：对量化具有惊人的鲁棒性。INT4 精度损失 < 1%。事实知识分布在许多权重上；量化噪声被平均掉。

**数学推理**：对量化更敏感。INT4 精度损失 3-8%。推理需要精确的数值计算；量化噪声会累积。

**代码生成**：中等敏感。INT4 精度损失 2-5%。语法保持完整；逻辑错误增加。

**创意写作**：低敏感。INT4 精度损失 < 2%。写作质量是主观的；微小的概率变化不会显著改变输出。

### 你应该记住的数字

- INT8：精度损失 < 0.5%，吞吐量提升 1.5-2 倍。
- INT4：精度损失 1-5%，吞吐量提升 2-4 倍。
- AWQ：保护 1% 的显著权重。
- GPTQ：需要 100-1000 个校准样本。
- GGUF：llama.cpp 格式；K-quant 变体。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 量化（Quantization） | "压缩模型" | 降低数值精度以换取速度 |
| INT8 | "8 位量化" | 安全默认值，精度损失 < 0.5% |
| INT4 | "4 位量化" | 激进选择，精度损失 1-5% |
| GPTQ | "基于 Hessian 矩阵" | 使用二阶导数信息的后训练量化 |
| AWQ | "保护重要权重" | 激活感知量化，保护显著权重 |
| SmoothQuant | "平滑异常值" | 将激活异常值转移到权重中 |
| GGUF | "llama.cpp 格式" | 捆绑量化权重 + 架构 + 元数据 |
| K-quant | "K 均值量化" | 非均匀量化变体（Q4_K_M 等） |
| 校准数据（Calibration data） | "量化样本" | 用于测量激活/权重敏感度的数据 |
| 显著权重（Salient weights） | "重要权重" | 对输出质量贡献最大的 1% 权重 |
