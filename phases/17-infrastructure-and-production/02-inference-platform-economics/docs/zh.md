# 推理平台经济学 — 每 Token 定价、预留吞吐量、Spot 推理与 TCO 模型

> LLM 推理定价在 2026 年已分化为五种模式。**按量付费（Pay-per-token）**：输入和输出 token 分开计费；输出 token 通常比输入 token 贵 3-5 倍（因为解码是内存带宽密集型且不可批处理）。**按请求付费（Pay-per-request）**：适合可预测的 token 消耗模式；供应商将 token 分布计入价格。**批量定价（Batch pricing）**：所有主要供应商对 24 小时内完成的异步批量作业提供 50% 折扣（第 17 阶段 · 15）。**预留吞吐量（Reserved throughput）**：承诺每月最低消费以换取 30-50% 折扣；适合可预测的稳态工作负载。**Spot 推理（Spot inference）**：使用空闲 GPU 容量，折扣 60-80%，但可能随时被抢占；适合容错性工作负载。成本模型必须考虑输入/输出 token 比例 — 一个以简短输入生成长篇输出的摘要应用，其成本结构完全不同于一个输入大量上下文但输出简短回答的 RAG（检索增强生成，Retrieval-Augmented Generation）应用。TCO（总拥有成本，Total Cost of Ownership）计算器必须包含：token 成本、网络出站费用、GPU 预留费用、缓存命中率折扣以及批量折扣。

## 学习目标

- 列举五种推理定价模式（按 token、按请求、批量、预留吞吐量、spot）并匹配到工作负载类型。
- 解释为什么输出 token 比输入 token 贵 3-5 倍，以及这对应用设计的影响。
- 构建一个包含 token 比例、缓存命中率和批量折扣的 TCO 模型。
- 计算从按需切换到预留 + spot 组合的盈亏平衡点。

## 问题

你的月度 LLM 账单是 $25,000。你按需支付每 token 费用。你的流量模式是：白天 12 小时稳定在 200 RPS（每秒请求数，Requests Per Second），夜间 12 小时稳定在 20 RPS。你为夜间流量支付了全价，而它本可以批量处理。你为白天流量支付了全价，而它本可以预留。你每年大约多付了 $120,000。

推理经济学不是关于 token 价格 — 而是关于将工作负载匹配到正确的定价模式。大多数团队在按需付费上停留太久，因为他们没有花一个小时来建模自己的流量形状。

## 概念

### 五种定价模式

**按量付费（Pay-per-token）**：输入和输出 token 分开计费。输出 token 贵 3-5 倍，因为解码是内存带宽密集型 — 每个 token 需要从 HBM（高带宽内存，High Bandwidth Memory）读取全部模型权重。输入处理（预填充，prefill）是计算密集型且可批处理，因此更便宜。适合：不可预测的流量、早期产品、低 RPS。

**按请求付费（Pay-per-request）**：供应商将 token 分布计入价格。适合：可预测的 token 消耗模式，其中每请求 token 数的方差较低。

**批量定价（Batch pricing）**：50% 折扣，24 小时完成 SLA（服务等级协议，Service Level Agreement）。所有主要供应商都提供。适合：内容生成流水线、文档分类、数据提取 — 任何能容忍 24 小时延迟的场景。详见第 17 阶段 · 15。

**预留吞吐量（Reserved throughput）**：承诺每月最低消费（例如 $5,000/月）以换取 30-50% 折扣。适合：可预测的稳态工作负载。类似于 AWS 的预留实例（Reserved Instances）。

**Spot 推理（Spot inference）**：使用空闲 GPU 容量，折扣 60-80%。可能随时被抢占（通常提前 30-60 秒通知）。适合：容错性工作负载、可重试的批量作业、非面向客户的推理。

### 输入/输出 token 比例决定成本结构

一个摘要应用：2K 输入 token，500 输出 token。输出 token 占 token 总数的 20%，但可能占成本的 50-60%（因为输出 token 更贵）。

一个 RAG 应用：8K 输入 token（检索到的文档），100 输出 token。输入 token 占 token 总数的 99%，但成本由输入定价主导。

在建模成本之前，先测量你的输入/输出比例。这个比例决定了哪种定价模式是最优的。

### TCO 模型

月度 TCO =（输入 token × 输入价格）+（输出 token × 输出价格）+ 网络出站费用 + GPU 预留费用 − 缓存命中折扣 − 批量折扣

缓存命中折扣（第 17 阶段 · 14）：缓存输入 token 的价格是未缓存输入的 10-20%。如果你的系统提示词是 4K token 且缓存命中率为 90%，输入成本会大幅下降。

批量折扣（第 17 阶段 · 15）：任何能容忍 24 小时延迟的工作负载立即获得 50% 折扣。

### 组合策略

2026 年的标准模式：白天稳态流量 → 预留吞吐量（30-50% 折扣）。夜间流量 → 批量 API（50% 折扣）。突发流量 → 按需或 spot。延迟关键路径 → 按需（无抢占风险）。

### 你应该记住的数字

- 输出 token 比输入 token 贵 3-5 倍。
- 批量折扣：50%，24 小时 SLA。
- 预留吞吐量折扣：30-50%。
- Spot 推理折扣：60-80%，可抢占。
- 缓存输入 token：比未缓存输入便宜 10-20 倍。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 按量付费（Pay-per-token） | "按使用付费" | 输入和输出 token 分开计费 |
| 批量定价（Batch pricing） | "异步折扣" | 50% 折扣，24 小时完成 |
| 预留吞吐量（Reserved throughput） | "承诺使用折扣" | 月度承诺换取 30-50% 折扣 |
| Spot 推理（Spot inference） | "空闲 GPU" | 60-80% 折扣，可抢占 |
| 输入/输出比例（Input/output ratio） | "token 混合" | 决定成本结构的关键比例 |
| TCO（总拥有成本） | "全成本" | Token + 网络 + GPU 预留 − 折扣 |
| 缓存命中折扣（Cache-hit discount） | "缓存节省" | 缓存输入 token 便宜 10-20 倍 |
| 抢占（Preemption） | "spot 被收回" | Spot 实例在 30-60 秒通知后被终止 |
