# AI Scientist v2 — Workshop 级自主研究

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）运行完整的研究循环：假设、代码、实验、图表、撰写、提交。它是第一个有生成论文通过 ICLR 2025 workshop 同行评审的系统。独立评估（Beel 等人）发现 42% 的实验因编码错误而失败，文献综述经常将已建立的概念错误标记为新颖。Sakana 自己的文档警告代码库执行 LLM 编写的代码，并建议使用 Docker 隔离。这幅图的两半都是重点。

**类型：** 学习
**语言：** Python（标准库，研究循环状态机玩具）
**前置条件：** 第 15 阶段 · 03（AlphaEvolve），第 15 阶段 · 04（DGM）
**时间：** 约 60 分钟

## 问题

研究是一个开放式任务。与 AlphaEvolve 的算法搜索或 DGM 的基准约束自修改不同，研究结果没有机器可检查的正确性标准。论文由审稿人评判，而不是单元测试。这使得循环更难闭合——如果闭合则更有价值，因为研究是复利进步所在。

AI Scientist v1（Sakana，2024）通过从人类编写的模板开始来闭合循环。LLM 在固定脚手架内填充实验。AI Scientist v2（Yamada 等人，2025）通过使用带有视觉语言模型批评循环的智能体树搜索来移除模板要求。系统生成想法、实现实验、生成图表、撰写论文，并根据审稿人反馈进行迭代。

同行评审结论：一篇 v2 生成的论文在 ICLR 2025 workshop 被接受（有披露）。独立评估结论：系统远不可靠。两者都是真实的。

## 概念

### 架构

1. **想法生成。** LLM 以主题和先前文献为条件提出研究想法。v1 使用模板；v2 使用在假设空间上的智能体搜索。
2. **新颖性检查。** 文献检索步骤检查想法是否已被发表。这是 Beel 等人评估发现错误标记的步骤——已建立的方法经常被分类为新颖。
3. **实验计划。** 智能体起草实验方案并编写代码。
4. **执行。** 代码在沙箱中运行。失败被反馈到重试循环中。在 Beel 等人的测量中，42% 的实验在此阶段因编码错误而失败。
5. **图表生成。** 视觉语言模型读取生成的图表并重写以提高清晰度。这是 v2 的关键技术添加。
6. **撰写。** LLM 起草论文，与内部审稿人迭代。
7. **可选：提交。** 论文提交到某个场所。

### Workshop 接受结果意味着什么

一篇 v2 生成的论文在 ICLR 2025 workshop 通过了同行评审。作者向程序委员会披露了论文的来源。接受是一个数据点；它不是声称系统"做研究"的许可。

重要背景：workshop 论文的门槛低于主会议论文。同行评审是有噪声的；在任何一天都有一小部分提交被接受。一次成功是概念验证，而不是可靠性声明。Nature 2026 论文记录了端到端循环，本身由人类研究人员共同撰写；它不是"系统写了一篇 Nature 论文"。

### 独立评估发现了什么

Beel 等人（arXiv:2502.14297）进行了外部评估。主要发现：

- **实验失败。** 42% 的实验因编码错误而失败（错误的导入、形状不匹配、未定义的变量）。重试循环捕获了一些，但不是全部。
- **新颖性错误标记。** 文献检索步骤经常将已建立的概念标记为新颖。这是研究的幻觉等价物。
- **呈现质量差距。** 视觉语言图表批评产生了出版级别的视觉效果，掩盖了底层的实验弱点。

最后一个发现对本阶段很重要。一个产生令人信服的输出而不做令人信服的研究的系统，比一个明显失败的系统更危险，而不是更安全。评估必须触及底层的声明，而不是止步于图表。

### 沙箱逃逸关切

Sakana 自己的仓库 README 警告：

> 由于本软件的性质（执行 LLM 生成的代码），我们无法保证安全。存在危险包、不受控制的网络访问和生成意外进程的风险。使用风险自负，并考虑 Docker 隔离。

这是未经验证领域中自主性的操作形态。LLM 编写代码；代码运行；代码可以做进程被允许做的任何事情。没有硬限制文件系统、网络和进程操作的沙箱，任何自主研究智能体都可以外泄数据、烧毁计算资源或重写自己。

AlphaEvolve 的沙箱故事更容易，因为其评估器是紧密的。AI Scientist v2 的循环以开放式目标运行开放式代码。这就是为什么它需要更强的隔离（Docker 最低要求；seccomp / gVisor 更优）以及在每次提交离开系统之前进行人工审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知失败 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元测试 + 基准测试 | 受评估器严谨性限制 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励黑客 |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、错误标记、润色掩盖弱点 |

v2 在三者中拥有最弱的自动评估器、最广的输出面和最短的到达公共产物的路径。操作控制（沙箱、审查、披露）承担了大部分安全工作。

## 使用它

`code/main.py` 将 v2 循环模拟为状态机：想法 → 新颖性检查 → 实验 → 图表 → 撰写 → 审查 → 接受或迭代。每个状态都有从 Beel 等人发现中提取的可配置失败概率。运行模拟器 N 个循环并计数：

- 有多少想法到达提交。
- 有多少提交会有润色后的论文隐藏的关键实验缺陷。
- 重试预算如何在质量与产出之间权衡。

## 交付它

`outputs/skill-ai-scientist-sandbox-review.md` 是一个双门审查清单，用于研究循环智能体产生的任何内容在离开沙箱之前。

## 练习

1. 使用默认参数运行 `code/main.py`。循环运行中有多大比例产生"干净"的论文？多大比例产生有实验失败缺陷但图表批评润色过的论文？

2. 默认值已经使用 Beel 等人的 42% / 25%。使用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 重新运行，然后使用 `--experiment-failure 0.60 --novelty-mislabel 0.40`。润色但有缺陷的份额在两次运行之间如何变化？

3. 阅读 Sakana 的 AI Scientist v2 仓库 README 中关于沙箱要求的内容。列出你将为多天自主运行应用的 Docker 之外的两项额外限制。

4. 阅读 Beel 等人第 4 节关于呈现质量差距的内容。设计一个额外的评估器，能够捕获看起来润色良好但实验有缺陷的论文。

5. 提出一个比"博士读每篇论文"更具可扩展性的研究智能体输出人工审查协议。确定瓶颈并围绕它设计。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| AI Scientist v1 | "Sakana 的模板化研究智能体" | 在固定脚手架内填充实验 |
| AI Scientist v2 | "无模板研究智能体" | 带有 VLM 图表批评的智能体树搜索 |
| 智能体树搜索 | "分支研究智能体" | 并行扩展多个实验计划；由内部批评者修剪 |
| 视觉语言批评 | "图表上的 VLM 润色" | 多模态模型读取图表并重写以提高清晰度 |
| 文献检索 | "新颖性检查" | 搜索先前工作以确认想法新颖性——有记录的错误标记 |
| 润色掩盖 | "漂亮的论文，破碎的研究" | 呈现质量超过实验质量；隐藏弱点 |
| 沙箱逃逸 | "LLM 代码突破" | 智能体执行的代码做了循环设计者未预期的事情 |
