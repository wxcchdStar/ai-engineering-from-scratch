# 机器学习中的统计学 (Statistics for Machine Learning)

> 统计学让你知道你的模型是真的有效还是只是走了狗屎运。

**类型：** 构建 (Build)
**语言：** Python
**前置要求：** 第一阶段，第 06 课（概率与分布）、第 07 课（贝叶斯定理）
**时间：** ~120 分钟

## 学习目标

- 从零计算描述性统计、Pearson/Spearman 相关性和协方差矩阵
- 执行假设检验（t 检验、卡方检验）并正确解释 p 值和置信区间
- 使用自助法 (Bootstrap) 重采样为任何度量构造置信区间，无需分布假设
- 使用效应量度量区分统计显著性和实际显著性

## 问题

你训练了两个模型。模型 A 在测试集上得分 0.87，模型 B 得分 0.89。你部署了模型 B。三周后，生产指标比以前更差。发生了什么？

模型 B 实际上并没有优于模型 A。0.02 的差异只是噪声。你的测试集太小，或者方差太高，或者两者都有。你把随机性包装成改进发布了。

这种事经常发生。Kaggle 排行榜震荡、无法复现的论文、仅基于几百个样本就宣布胜者的 A/B 测试——根本原因永远是一样的：有人跳过了统计学。

统计学给你区分信号和噪声的工具。它告诉你差异何时是真实的、你应该有多大的信心、以及在相信一个结果之前你需要多少数据。每个 ML 流程、每个模型比较、每个实验都需要统计学。没有它，你就是在猜。

## 概念

### 描述性统计：总结你的数据

在你建模任何东西之前，你需要知道数据长什么样。描述性统计将数据集压缩为几个捕获其形状的数字。

**集中趋势的度量**回答"中心在哪里？"

```
均值 (Mean)：    所有值之和 / 计数
                mu = (1/n) * sum(x_i)

中位数 (Median)：排序后的中间值
                对异常值鲁棒。如果你有 [1, 2, 3, 4, 1000]，均值是 202
                但中位数是 3。

众数 (Mode)：    最频繁出现的值
                对分类数据有用。对连续数据很少有意义。
```

均值是平衡点，中位数是半程标记。当它们偏离时，你的分布是偏斜的。收入分布有均值 >> 中位数（亿万富翁造成的右偏）。训练过程中的损失分布通常均值 << 中位数（简单样本造成的左偏）。

**离散度的度量**回答"数据有多分散？"

```
方差 (Variance)：          与均值的平方偏差的平均值
                           sigma^2 = (1/n) * sum((x_i - mu)^2)

标准差 (Standard Deviation)：方差的平方根
                              sigma = sqrt(sigma^2)
                              与数据单位相同，因此更易解释。

极差 (Range)：             max - min
                          对异常值敏感。几乎从不单独使用。

四分位距 (IQR)：           Q3 - Q1（四分位距）
                           中间 50% 数据的范围。
                           对异常值鲁棒。用于箱线图和异常检测。
```

**百分位数 (Percentiles)** 将排序数据分为 100 等份。第 25 百分位数 (Q1) 表示 25% 的值低于此点。第 50 百分位数是中位数，第 75 百分位数是 Q3。

```
对于延迟监控：
  P50 = 中位延迟          （典型用户体验）
  P95 = 第 95 百分位      （不好但不是最坏情况）
  P99 = 第 99 百分位      （长尾延迟，通常为中位数 10 倍）
```

在 ML 中，你关心推理延迟、预测置信度分布和理解误差分布的百分位数。平均误差低但 P99 误差很高的模型对安全关键应用可能毫无用处。

**样本 vs 总体统计量。** 从样本计算方差时，分母用 (n-1) 而非 n。这是贝塞尔校正 (Bessel's Correction)，它补偿了样本均值不是真实总体均值这一事实。用 n 做分母会系统性地低估真实方差，用 (n-1) 则估计是无偏的。

```
总体方差：sigma^2 = (1/N) * sum((x_i - mu)^2)
样本方差：s^2      = (1/(n-1)) * sum((x_i - x_bar)^2)
```

实践中：如果 n 很大（数千个样本），差异可忽略；如果 n 很小（几十个样本），差异就很重要。

### 相关性：变量如何一起变动

相关性衡量两个变量之间线性关系的强度和方向。

**Pearson 相关系数**衡量线性关联：

```
r = sum((x_i - x_bar)(y_i - y_bar)) / (n * s_x * s_y)

r = +1：  完美正线性关系
r = -1：  完美负线性关系
r =  0：  无线性关系（但可能存在非线性关系！）

范围：[-1, 1]
```

Pearson 假设关系是线性的且两个变量大致服从正态分布。它对异常值敏感——一个极端点就能将 r 从 0.1 拖到 0.9。

**Spearman 秩相关系数**衡量单调关联：

```
1. 将每个值替换为其秩（1, 2, 3, ...）
2. 对秩计算 Pearson 相关系数

Spearman 捕获任何单调关系，不仅仅是线性。
如果 y = x^3，Pearson 给出 r < 1 但 Spearman 给出 rho = 1。
```

**何时使用每种方法：**

```
Pearson：   两个变量都是连续且大致正态的。
           你特别关心线性关系。
           没有极端异常值。

Spearman：  有序数据（排名、评分）。
           数据不服从正态分布。
           你怀疑有单调但非线性的关系。
           存在异常值。
```

**黄金法则：** 相关性不意味着因果关系。冰淇淋销量和溺水死亡人数相关，因为两者都在夏季增加。模型的精度和参数数量相关，但增加参数并不会自动提高精度（参见：过拟合）。

### 协方差矩阵

两个变量之间的协方差衡量它们如何一起变动：

```
Cov(X, Y) = (1/n) * sum((x_i - x_bar)(y_i - y_bar))

Cov(X, Y) > 0：  X 和 Y 倾向于一起增加
Cov(X, Y) < 0：  当 X 增加时，Y 倾向于减少
Cov(X, Y) = 0：  无线性共变关系
```

对于 d 个特征，协方差矩阵 C 是一个 d x d 矩阵，其中 C[i][j] = Cov(特征_i, 特征_j)。对角线项 C[i][i] 是每个特征的方差。

```
C = | Var(x1)      Cov(x1,x2)  Cov(x1,x3) |
    | Cov(x2,x1)  Var(x2)      Cov(x2,x3) |
    | Cov(x3,x1)  Cov(x3,x2)  Var(x3)     |

性质：
  - 对称：C[i][j] = C[j][i]
  - 半正定：所有特征值 >= 0
  - 对角线 = 方差
  - 非对角线 = 协方差
```

**与 PCA 的联系。** PCA 对协方差矩阵进行特征分解。特征向量是主成分（最大方差方向），特征值告诉你每个成分捕获了多少方差。这正是第 10 课讲的内容，但现在你明白了为什么协方差矩阵是要分解的正确对象：它编码了数据中所有成对的线性关系。

**与相关性的联系。** 相关矩阵是标准化变量（每个变量除以其标准差）的协方差矩阵。相关性将协方差归一化使所有值落在 [-1, 1] 内。

### 假设检验

假设检验是在不确定性下做决策的框架。你从一个声明开始，收集数据，判断数据是否与声明一致。

**设定：**

```
原假设 (Null Hypothesis, H0)：      默认假设，通常是"没有效果"
备择假设 (Alternative Hypothesis, H1)：你试图证明的东西

示例：
  H0：模型 A 和模型 B 有相同的精度
  H1：模型 B 比模型 A 精度更高
```

**p 值** 是在假设 H0 为真的前提下，看到与观察数据同等极端或更极端数据的概率。它不是 H0 为真的概率——这是统计学中最常见的误解。

```
p 值 = P(数据这么极端 | H0 为真)

如果 p 值 < alpha（通常 0.05）：
    拒绝 H0。结果是"统计显著的 (statistically significant)"。
如果 p 值 >= alpha：
    不能拒绝 H0。你没有足够的证据。
    这并不意味着 H0 为真。
```

**置信区间 (Confidence Interval)** 给出一个参数的可信值范围：

```
均值的 95% 置信区间：
    x_bar +/- z * (s / sqrt(n))

其中 z = 1.96 对应 95% 置信度

解释：如果你多次重复这个实验，95% 的计算区间将包含真实均值。
这并不意味着真实均值有 95% 的概率在这个特定区间内。
```

置信区间的宽度告诉你精度。宽区间意味着高不确定性，窄区间意味着你的估计精确（但如果数据有偏差，不一定是准确的）。

### t 检验

t 检验比较均值。有几种变体。

**单样本 t 检验：** 总体均值是否与假设值不同？

```
t = (x_bar - mu_0) / (s / sqrt(n))

自由度 = n - 1
```

**双样本 t 检验（独立）：** 两组均值是否不同？

```
t = (x_bar_1 - x_bar_2) / sqrt(s1^2/n1 + s2^2/n2)

这是 Welch's t 检验，不假设方差相等。
除非有特定理由假设方差相等，否则始终使用 Welch 版本。
```

**配对 t 检验：** 当测量成对出现时（同一模型在相同数据划分上评估）：

```
计算 d_i = x_i - y_i 对每对
然后在 d_i 值上对 mu_0 = 0 运行单样本 t 检验
```

在 ML 中，配对 t 检验很常见：你在相同的 10 个交叉验证折上运行两个模型，成对比较它们的分数。

### 卡方检验

卡方检验检查观测频率是否与期望频率匹配。适用于分类数据。

```
chi^2 = sum((观测值 - 期望值)^2 / 期望值)

示例：一个语言模型的输出分布是否与训练分布
在各类别上匹配？

类别       观测     期望
正面        120      100
负面         80      100
chi^2 = (120-100)^2/100 + (80-100)^2/100 = 4 + 4 = 8

自由度为 1，chi^2 = 8 给出 p < 0.005。
差异是显著的。
```

### ML 模型的 A/B 测试

ML 中的 A/B 测试不同于网页 A/B 测试。模型比较有特定挑战：

```
1. 相同测试集：    两个模型必须在相同的数据上评估。
                   不同的测试集使比较毫无意义。

2. 多度量：       单独看准确率是不够的。你需要精确率、
                   召回率、F1、延迟和公平性度量。

3. 方差：         使用交叉验证或自助法来估计
                   每个度量的方差，而不仅仅是点估计。

4. 数据泄露：     如果测试集在模型选择期间被使用过，
                   你的比较就有偏差。保留一个最终测试集。
```

**流程：**

```
1. 定义你的度量和显著性水平（alpha = 0.05）
2. 在相同的 k 折交叉验证划分上运行两个模型
3. 收集配对分数：[(a1, b1), (a2, b2), ..., (ak, bk)]
4. 计算差异：d_i = b_i - a_i
5. 对差异运行配对 t 检验
6. 检查：均值差异是否显著不为零？
7. 计算均值差异的置信区间
8. 计算效应量（Cohen's d）来判断实际显著性
```

### 统计显著性 vs 实际显著性

一个结果可能在统计上显著但在实际上毫无意义。有了足够的数据，即使是微小的差异也会变得统计显著。

```
示例：
  模型 A 准确率：0.9234
  模型 B 准确率：0.9237
  n = 1,000,000 个测试样本
  p 值 = 0.001

统计显著？是的。
实际显著？0.03% 的改进不值得部署新模型的工程成本。
```

**效应量 (Effect Size)** 量化差异有多大，与样本量无关：

```
Cohen's d = (均值1 - 均值2) / 合并标准差

d = 0.2：  小效应
d = 0.5：  中等效应
d = 0.8：  大效应
```

始终同时报告 p 值和效应量。p 值告诉你差异是否真实，效应量告诉你它是否重要。

### 多重比较问题

当你检验很多假设时，有些会碰巧"显著"。如果你在 alpha = 0.05 下检验 20 个东西，即使没有任何真实效应，你也预期有 1 个假阳性。

```
P（至少一个假阳性）= 1 - (1 - alpha)^m

m = 20 次检验，alpha = 0.05：
P(假阳性) = 1 - 0.95^20 = 0.64

你有 64% 的概率得到至少一个假阳性。
```

**Bonferroni 校正：** 将 alpha 除以检验次数。

```
调整后 alpha = alpha / m = 0.05 / 20 = 0.0025

只有当 p 值 < 0.0025 时才拒绝 H0。
保守但简单。当检验独立时有效。
```

在 ML 中，这在你跨多个度量比较模型、测试许多超参数配置或在多个数据集上评估时很重要。

### 自助法 (Bootstrap Methods)

自助法通过对你的数据进行有放回重采样来估计统计量的抽样分布。不需要对底层分布做任何假设。

**算法：**

```
1. 你有 n 个数据点
2. 有放回地抽取 n 个样本（某些点出现多次，
   某些点根本不出现）
3. 在这个自助样本上计算你的统计量
4. 重复 B 次（通常 B = 1000 到 10000）
5. 自助统计量的分布近似于
   抽样分布
```

**自助法置信区间（百分位方法）：**

```
对 B 个自助统计量排序
95% CI = [2.5 百分位, 97.5 百分位]
```

**为什么自助法对 ML 重要：**

```
- 测试集准确率是一个点估计。自助法给你
  置信区间。
- 你不能假设度量分布是正态的（尤其对
  AUC、F1、precision at k 等）。
- 自助法适用于任何统计量：中位数、两个均值的比率、
  两个模型 AUC 的差异。
- 不需要封闭形式公式。
```

**用于模型比较的自助法：**

```
1. 你有模型 A 和模型 B 在相同测试集上的预测
2. 对每次自助迭代：
   a. 有放回地重采样测试索引
   b. 在重采样集上计算 metric_A 和 metric_B
   c. 存储 diff = metric_B - metric_A
3. 差异的 95% CI：
   [diff 的 2.5 百分位, diff 的 97.5 百分位]
4. 如果 CI 不包含 0，差异是显著的
```

这比配对 t 检验更鲁棒，因为它不做分布假设。

### 参数检验 vs 非参数检验

**参数检验** 假设特定分布（通常是正态）：

```
t 检验：          假设数据服从正态分布（或大 n 时由 CLT 保证）
ANOVA：           假设正态性和等方差
Pearson r：       假设双变量正态
```

**非参数检验** 不做分布假设：

```
Mann-Whitney U：     比较两组（替代独立 t 检验）
Wilcoxon 符号秩：    比较配对数据（替代配对 t 检验）
Spearman rho：       对秩做相关（替代 Pearson）
Kruskal-Wallis：     比较多组（替代 ANOVA）
```

**何时使用非参数：**

```
- 小样本量（n < 30）且数据明显非正态
- 有序数据（评分、排名）
- 存在无法移除的严重异常值
- 偏斜分布
```

**何时使用参数：**

```
- 大样本量（CLT 使检验统计量近似正态）
- 数据大致对称且无极端异常值
- 具有更高的统计功效（更擅长检测真实差异）
```

在 ML 实验中，你通常有小 n（5 或 10 折交叉验证），因此威尔科克森符号秩检验等非参数检验通常比 t 检验更合适。

### 中心极限定理 (Central Limit Theorem)：实际含义

CLT 说，当 n 增长时，样本均值的分布趋近正态分布，无论底层总体分布如何。

```
如果 X_1, X_2, ..., X_n 独立同分布，均值为 mu，方差为 sigma^2：

    X_bar ~ Normal(mu, sigma^2 / n)    当 n -> 无穷时

大多数情况下 n >= 30 就有效。
对严重偏斜的分布，你可能需要 n >= 100。
```

**为什么这对 ML 重要：**

```
1. 为聚合度量的置信区间和 t 检验提供了理论依据
2. 解释了为什么对交叉验证折取平均能得到稳定估计，
   即使单个折之间差异很大
3. 小批量梯度下降之所以有效，是因为一个批次的
   平均梯度近似于真实梯度（CLT 在起作用）
4. 集成方法：对多个模型的预测取平均得到比任何
   单一模型更稳定的输出
```

**CLT 不做的事情：**

```
- 不使你的数据变正态。它使样本的均值变正态。
- 对具有无穷方差的厚尾分布无效
  （柯西分布 Cauchy distribution）。
- 不适用于依赖数据（未经校正的时间序列）。
```

### ML 论文中的常见统计错误

1. **在训练集上测试。** 必然导致过拟合。始终保留模型在训练期间从未见过的数据。

2. **没有置信区间。** 只报告一个准确率数字而不含不确定性使结果不可复现也无法验证。

3. **忽略多重比较。** 测试 50 种配置并报告最佳结果而不做校正，会夸大假阳性率。

4. **混淆统计显著性和实际显著性。** 对 0.01% 的准确率改进 p 值为 0.001，这没有意义。

5. **在不平衡数据上使用准确率。** 在 99% 为负类的数据集上得到 99% 准确率，意味着模型什么都没学到。使用精确率、召回率、F1 或 AUC。

6. **挑选度量。** 只报告你的模型胜出的度量。诚实的评估应报告所有相关度量。

7. **在训练/测试划分间泄露信息。** 在划分前做归一化，或用未来的数据预测过去。

8. **小测试集且没有方差估计。** 在 100 个样本上评估并声称有 2% 的改进——这是噪声，不是信号。

9. **假设独立性但数据并不独立。** 来自同一患者的医学图像、来自同一文档的多个句子。组内观察是相关的。

10. **P 值操控 (P-hacking)。** 尝试不同的检验、子集或排除标准直到得到 p < 0.05。结果是搜索过程的人造产物。

6. **统计 vs 实际显著性演示**（展示大数据量如何使一切变得"显著"）

全部从零构建，仅使用 `math` 和 `random`。不用 numpy，不用 scipy。

## 关键术语

| 术语 | 定义 |
|------|------|
| 均值 (Mean) | 值之和除以计数。对异常值敏感。 |
| 中位数 (Median) | 排序数据的中间值。对异常值鲁棒。 |
| 标准差 (Standard Deviation) | 方差的平方根。以原始单位衡量离散度。 |
| 百分位数 (Percentile) | 给定百分比的数据低于此值。 |
| IQR | 四分位距。Q3 减 Q1。中间 50% 的离散度。 |
| Pearson 相关系数 | 衡量两个变量之间的线性关联。范围 [-1, 1]。 |
| Spearman 相关系数 | 使用秩衡量单调关联。 |
| 协方差矩阵 | 所有特征之间成对协方差的矩阵。 |
| 原假设 (Null Hypothesis) | 默认假设——无效应或无差异。 |
| p 值 | 在原假设为真的前提下，得到与观测数据同等极端或更极端数据的概率。 |
| 置信区间 (Confidence Interval) | 给定置信水平下参数的合理值范围。 |
| t 检验 | 检验均值是否显著不同。使用 t 分布。 |
| 卡方检验 | 检验观测频率是否与期望频率不同。 |
| 效应量 (Effect Size) | 差异的大小，与样本量无关。常用 Cohen's d。 |
| Bonferroni 校正 | 将显著性阈值除以检验次数以控制假阳性。 |
| 自助法 (Bootstrap) | 有放回重采样以估计抽样分布。 |
| 一类错误 (Type I Error) | 假阳性。H0 为真时拒绝 H0。 |
| 二类错误 (Type II Error) | 假阴性。H0 为假时未能拒绝 H0。 |
| 统计功效 (Statistical Power) | 正确拒绝错误 H0 的概率。功效 = 1 - 二类错误率。 |
| 中心极限定理 (CLT) | 随着样本量增长，样本均值收敛到正态分布。 |
| 参数检验 (Parametric Test) | 假设数据服从特定分布（通常是正态分布）。 |
| 非参数检验 (Non-parametric Test) | 不做分布假设。基于秩或符号工作。 |
