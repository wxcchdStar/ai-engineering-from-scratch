# 范数与距离 (Norms and Distances)

> 你的距离函数定义了什么叫"相似"。选错了，下游一切都会出问题。

**类型：** 构建 (Build)
**语言：** Python
**前置要求：** 第一阶段，第 01 课（线性代数直觉）、第 02 课（向量、矩阵与运算）

## 学习目标

- 从零实现 L1、L2、余弦、马氏距离 (Mahalanobis)、Jaccard 和编辑距离函数
- 为给定 ML 任务选择合适的距离度量并解释为什么其他选择会失败
- 将 L1 和 L2 范数与 LASSO 和 Ridge 正则化及其几何约束区域联系起来
- 演示同一数据集在不同度量下如何产生不同的最近邻

## 问题

你有两个向量。可能是词嵌入，可能是用户画像，可能是像素数组。你需要知道：它们有多接近？

答案完全取决于你选哪个距离函数。两个数据点可能在一种度量下是最近邻，在另一种度量下相距很远。你的 KNN 分类器、推荐引擎、向量数据库、聚类算法、损失函数——都依赖于这个选择。选错了，模型就会优化错误的东西。

没有放之四海皆准的最佳距离。L2 适用于空间数据。余弦相似度主导 NLP。Jaccard 处理集合。编辑距离处理字符串。马氏距离 (Mahalanobis) 考虑相关性。Wasserstein 移动概率质量。每种距离编码了对"相似"含义的不同假设。

本课从零构建每个主要距离函数，展示每种方法在何时是正确工具，并演示同一数据在不同度量下如何产生完全不同的最近邻。

## 概念

### 范数：测量向量大小

范数测量向量的"大小"。两个向量之间的每种距离函数都可以写成它们差值的范数：d(a, b) = ||a - b||。所以理解范数就是理解距离。

### L1 范数（曼哈顿距离 Manhattan Distance）

L1 范数是所有分量绝对值之和。

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

它叫曼哈顿距离，因为它测量的是在城市网格上只能沿轴线走时的行走距离——没有对角线。

```
点 A = (1, 1)
点 B = (4, 5)

L1 距离 = |4-1| + |5-1| = 3 + 4 = 7

在网格上，你向东走 3 个街区，向北走 4 个街区。
```

何时使用 L1：
- 高维稀疏数据（文本特征、独热编码）
- 当你想要对异常值具有鲁棒性时（单个巨大差异不会主导结果）
- 特征选择问题（L1 正则化促进稀疏性）

与 L1 正则化（Lasso）的联系：将 ||w||_1 加到损失函数中，惩罚的是权重绝对值之和。这会将小权重恰好推到零，实现自动特征选择。L1 惩罚在权重空间中产生菱形约束区域，菱形的角位于轴上，即某些权重为零的位置。

与损失函数的联系：平均绝对误差 (MAE) 是预测与目标之间的平均 L1 距离。它对所有误差线性惩罚，使其相比 MSE 对异常值更鲁棒。

### L2 范数（欧氏距离 Euclidean Distance）

L2 范数是直线距离——各分量平方和的平方根。

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

这就是你在几何课上学到的距离——n 维空间中的勾股定理。

```
点 A = (1, 1)
点 B = (4, 5)

L2 距离 = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

直线，穿过网格的对角线。
```

何时使用 L2：
- 低到中维度的连续数据
- 当特征尺度可比较时
- 物理距离（空间数据、传感器读数）
- 像素级别的图像相似度

与 L2 正则化（Ridge）的联系：将 ||w||_2^2 加到损失函数中，惩罚的是大权重。与 L1 不同，它不会将权重推到零，而是将所有权重按比例向零收缩。L2 惩罚产生圆形约束区域，所以轴上没有角——权重变小但很少恰好为零。

与损失函数的联系：均方误差 (MSE) 是 L2 距离平方的平均值。平方对大误差的惩罚远重于小误差。

```
MAE（L1 损失）：|y - y_hat|           线性惩罚，对异常值鲁棒
MSE（L2 损失）：(y - y_hat)^2          二次惩罚，对异常值敏感
```

### Lp 范数：通用族系

L1 和 L2 是 Lp 范数的特例：

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

不同的 p 值产生不同形状的"单位球"（所有距离原点为 1 的点的集合）：

```
p=1:    菱形            （角在轴上）
p=2:    圆/球           （通常的圆球）
p=3:    超椭圆          （圆角正方形）
p=inf:  正方形/超立方体  （沿轴的平面）
```

### L-无穷范数（切比雪夫距离 Chebyshev Distance）

当 p 趋于无穷时，Lp 范数收敛为最大绝对值分量。

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

两点之间的距离由它们差异最大的那个维度决定，其他所有维度被忽略。

```
点 A = (1, 1)
点 B = (4, 5)

L-inf 距离 = max(|4-1|, |5-1|) = max(3, 4) = 4
```

何时使用 L-无穷：
- 当任何单一维度的最坏情况偏差很重要时
- 棋盘游戏（国际象棋中的王以 L-无穷方式移动：向任意方向走一步花费 1）
- 制造公差（每个维度都必须在规格范围内）

### 余弦相似度 (Cosine Similarity) 和余弦距离 (Cosine Distance)

余弦相似度测量两个向量之间的角度，忽略它们的长度。

```
cos_sim(a, b) = (a . b) / (||a||_2 * ||b||_2)
```

范围从 -1（方向相反）到 +1（方向相同）。垂直向量的余弦相似度为 0。

余弦距离将其转换为距离：cosine_distance = 1 - cosine_similarity。范围从 0（方向相同）到 2（方向相反）。

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

为什么余弦在 NLP 和嵌入中占主导地位：在文本中，文档长度不应影响相似度。一篇关于猫的文档如果长度是另一篇的两倍，它们仍然应该"相似"。余弦相似度忽略长度（大小），只关心方向。两篇具有相同词分布但长度不同的文档指向相同方向，得到余弦相似度 1.0。

何时使用余弦相似度：
- 文本相似度（TF-IDF 向量、词嵌入、句子嵌入）
- 任何长度是噪声、方向是信号的领域
- 推荐系统（用户偏好向量）
- 嵌入搜索（向量数据库几乎总是使用余弦或点积）

### 点积相似度 vs 余弦相似度

两个向量的点积是：

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(角度)
```

余弦相似度是用两个长度归一化后的点积。当两个向量都已经单位归一化（长度 = 1）时，点积和余弦相似度完全相同。

```
如果 ||a|| = 1 且 ||b|| = 1：
    a . b = cos(a 和 b 之间的角度)
```

当它们不同时：点积包含长度信息。具有更大长度的向量获得更高的点积分数。这在某些检索系统中很重要，你希望"热门"物品排名更高——长度充当隐式的质量或重要性信号。

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

两者在方向上一致，但点积也反映了长度。
```

实践中：
- 当你想要纯粹的方向相似度时，使用余弦相似度
- 当长度携带有意义的信息时，使用点积
- 许多向量数据库（Pinecone、Weaviate、Qdrant）让你选择
- 如果你的嵌入已经是 L2 归一化的，选择哪个都不重要

### 马氏距离 (Mahalanobis Distance)

欧氏距离将所有维度同等对待。但如果你的特征是相关的或尺度不同，L2 会产生误导结果。

马氏距离考虑了数据的协方差结构。

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

其中 S 是数据的协方差矩阵。

直观理解：马氏距离首先对数据做去相关和归一化（白化 whitening），然后在变换后的空间中计算 L2 距离。如果 S 是单位矩阵（不相关、单位方差的特征），马氏距离退化为欧氏距离。

```
示例：身高和体重是相关的。
某人 6'2" 180 磅并不异常。
某人 5'0" 180 磅是异常的。

欧氏距离可能说它们离均值一样远。
马氏距离正确地将第二个标识为异常值，
因为它考虑了身高-体重的相关性。
```

何时使用马氏距离：
- 异常检测（马氏距离离均值大的点是异常值）
- 当特征有不同尺度和相关性时的分类
- 当你有足够数据估计可靠的协方差矩阵时
- 制造质量控制（多变量过程监控）

### Jaccard 相似度（用于集合）

Jaccard 相似度测量两个集合之间的重叠。

```
J(A, B) = |A ∩ B| / |A ∪ B|
```

范围从 0（没有重叠）到 1（集合相同）。Jaccard 距离 = 1 - Jaccard 相似度。

```
A = {cat, dog, fish}
B = {cat, bird, fish, snake}

交集 = {cat, fish}                大小 = 2
并集 = {cat, dog, fish, bird, snake}  大小 = 5

Jaccard 相似度 = 2/5 = 0.4
Jaccard 距离 = 0.6
```

何时使用 Jaccard：
- 比较标签、类别或特征的集合
- 基于词的存在（而非频率）的文档相似度
- 近似重复检测（Jaccard 的 MinHash 近似）
- 比较二进制特征向量（存在/不存在数据）
- 评估分割模型（交并比 IoU = Jaccard）

### 编辑距离（莱文斯坦距离 Levenshtein Distance）

编辑距离计算将一个字符串转换为另一个字符串所需的最小单字符操作数。操作包括：插入、删除或替换。

```
"kitten" -> "sitting"

kitten -> sitten  （将 k 替换为 s）
sitten -> sittin  （将 e 替换为 i）
sittin -> sitting （插入 g）

编辑距离 = 3
```

使用动态规划计算。填充一个矩阵，其中 (i, j) 项是字符串 A 的前 i 个字符和字符串 B 的前 j 个字符之间的编辑距离。

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

何时使用编辑距离：
- 拼写检查和修正
- DNA 序列比对（使用加权操作）
- 模糊字符串匹配
- 脏文本数据的去重

### KL 散度（不是距离，但被当作距离使用）

KL 散度测量一个概率分布与另一个的差异。在第 09 课中详细讲过，但它在此处的讨论中有一席之地，因为人们把它当作"距离"用，尽管它不是。

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

关键性质：KL 散度不是对称的。

```
D_KL(P || Q) != D_KL(Q || P)
```

这意味着它不满足距离度量的基本要求。它也不满足三角不等式。它是散度 (divergence)，不是距离。

前向 KL (D_KL(P || Q)) 是"均值追寻"的：Q 试图覆盖 P 的所有模态。
反向 KL (D_KL(Q || P)) 是"模态追寻"的：Q 聚焦于 P 的单一模态。

当你看到 KL 散度时：
- VAE（ELBO 中的 KL 项将潜在分布推向先验）
- 知识蒸馏（学生试图匹配教师的分布）
- RLHF（KL 惩罚保持微调模型接近基础模型）
- 策略梯度方法（约束策略更新）

### Wasserstein 距离（推土机距离 Earth Mover's Distance）

Wasserstein 距离测量将一个概率分布转换为另一个所需的最小"功"。可以把它想象成：如果一个分布是一堆土，另一个是一个坑，你需要移动多少土、移动多远？

```
W(P, Q) = inf over all transport plans gamma of E[d(x, y)]
```

对于一维分布，它简化为累积分布函数绝对差的积分：

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

为什么 Wasserstein 重要：
- 它是真正的度量（对称、满足三角不等式）
- 即使分布没有重叠，它也提供梯度（KL 散度会趋于无穷）
- 这一属性使其成为 Wasserstein GAN（WGAN）的核心，解决了原始 GAN 的训练不稳定问题

```
没有重叠的分布：

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL 散度：无穷（零的对数）
Wasserstein：4（将所有质量移动 4 个 bin）

Wasserstein 给出有意义的梯度，KL 不能。
```

何时使用 Wasserstein：
- GAN 训练（WGAN、WGAN-GP）
- 比较可能没有重叠的分布
- 最优传输问题
- 图像检索（比较颜色直方图）

### 为什么不同任务需要不同距离

| 任务 | 最佳距离 | 原因 |
|------|---------|------|
| 文本相似度 | 余弦 | 长度是噪声，方向是含义 |
| 图像像素比较 | L2 | 空间关系重要，特征有可比尺度 |
| 稀疏高维特征 | L1 | 鲁棒，不会放大罕见的大差异 |
| 集合重叠（标签、类别） | Jaccard | 数据本质上是集合，不是向量 |
| 字符串匹配 | 编辑距离 | 操作映射到人类的编辑直觉 |
| 异常检测 | 马氏距离 | 考虑特征相关性和尺度 |
| 比较分布 | KL 散度 | 测量用 Q 替代 P 损失的信息 |
| GAN 训练 | Wasserstein | 即使分布不重叠也提供梯度 |
| 嵌入（向量数据库） | 余弦或点积 | 嵌入被训练为在方向上编码含义 |
| 推荐 | 点积 | 长度可以编码流行度或置信度 |
| DNA 序列 | 加权编辑距离 | 替换成本因核苷酸对而异 |
| 制造质量控制 | L-无穷 | 任何维度的最坏情况偏差都重要 |

### 与损失函数的联系

损失函数是应用于预测与目标之间的距离函数。

```
损失函数           使用的距离          行为
MSE              L2 平方              对大误差惩罚很重
MAE              L1                   对所有误差平等惩罚
Huber 损失       对大误差用 L1，         两全其美：对异常值鲁棒，
                 对小误差用 L2          在零点附近梯度平滑
交叉熵           KL 散度               测量分布不匹配
合页损失         max(0, margin - d)    仅在低于间隔时惩罚
三元组损失       L2（通常）            拉近正样本，推远负样本
对比损失         L2                   相似对接近，不相似对
                                      超出间隔
```

### 与正则化的联系

正则化将权重的范数惩罚加到损失函数上。

```
L1 正则化（Lasso）：    loss + lambda * ||w||_1
  -> 稀疏权重。部分权重恰好变为零。
  -> 自动特征选择。
  -> 解有角点（在零点不可微）。

L2 正则化（Ridge）：    loss + lambda * ||w||_2^2
  -> 小权重。所有权重向零收缩。
  -> 无特征选择（没有权重恰好为零）。
  -> 解处处光滑。

弹性网络 (Elastic Net)： loss + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> 结合 L1 的稀疏性和 L2 的稳定性。
  -> 相关特征组被一起保留或丢弃。
```

为什么 L1 产生稀疏性而 L2 不产生：想象 2D 权重空间中的约束区域。L1 是菱形，L2 是圆。损失函数的等值线（椭圆）最可能在菱形的角处碰到约束，那里一个权重为零。它们在圆的平滑点处碰到约束，那里两个权重都非零。

### 最近邻搜索

每种距离函数都隐含了一个最近邻搜索问题：给定一个查询点，在数据集中找到最接近的点。

精确最近邻搜索在包含 n 个 d 维数据点的数据集中每个查询耗时 O(n * d)。对于大数据集，这太慢了。

近似最近邻算法 (Approximate Nearest Neighbor, ANN) 以少量精度换取巨大的速度提升：

```
算法             方法                          使用者
KD 树            轴对齐空间划分                  scikit-learn（低维）
Ball 树          嵌套超球体                      scikit-learn（中维）
LSH              随机哈希投影                    近似重复检测
HNSW             分层可导航小世界图               FAISS, Qdrant, Weaviate
IVF              基于聚类的倒排索引               FAISS（十亿级）
乘积量化          压缩向量，在压缩空间中搜索        FAISS（内存受限）
```

HNSW（分层可导航小世界图 Hierarchical Navigable Small World）是现代向量数据库中的主导算法。它构建一个多层图，每个节点连接到其近似最近邻。搜索从顶层开始（稀疏、长跳跃），降到底层（密集、短跳跃）。

## 使用

最常见的实际用途：在向量数据库中查找相似项。

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"Top 5 most similar to item 0: {top_k}")
print(f"Similarities: {similarities[top_k]}")
```

当你调用 `model.encode(text)` 然后搜索向量数据库时，幕后发生的就是这个。嵌入模型将文本映射为向量，向量数据库使用 ANN 算法计算你的查询向量与每个存储向量之间的余弦相似度（或点积），避免检查所有向量。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 范数 (Norm) | "向量的大小" | 将向量映射为非负标量的函数，满足三角不等式、绝对齐次性和仅对零向量为零 |
| L1 范数 | "曼哈顿距离" | 分量绝对值之和。在优化中产生稀疏性。对异常值鲁棒 |
| L2 范数 | "欧氏距离" | 分量平方和的平方根。欧氏空间中的直线距离 |
| Lp 范数 | "广义范数" | 分量绝对值 p 次方之和的 p 次方根。L1 和 L2 是特例 |
| L-无穷范数 | "最大范数"或"切比雪夫距离" | 最大绝对值分量。p 趋于无穷时 Lp 的极限 |
| 余弦相似度 | "向量间的角度" | 用两个长度归一化后的点积。范围 -1 到 +1。忽略向量长度 |
| 余弦距离 | "1 减余弦相似度" | 将余弦相似度转换为距离。范围 0 到 2 |
| 点积 | "未归一化的余弦" | 分量逐项乘积之和。等于余弦相似度乘以两个长度 |
| 马氏距离 | "考虑相关性的距离" | 在已白化（使用数据协方差矩阵去相关和归一化）的空间中的 L2 距离 |
| Jaccard 相似度 | "集合重叠" | 交集大小除以并集大小。用于集合，非向量 |
| 编辑距离 | "莱文斯坦距离" | 将一个字符串转换为另一个所需的最少插入、删除和替换次数 |
| KL 散度 | "分布间的距离" | 不是真正的距离（不对称）。测量用 Q 编码 P 所需的额外比特数 |
| Wasserstein 距离 | "推土机距离" | 将一个分布的质量搬运到另一个所需的最小功。是真正的度量 |
| 近似最近邻 (ANN) | "ANN 搜索" | 比精确搜索快得多地找到近似最近点的算法（HNSW、LSH、IVF） |
| HNSW | "向量数据库算法" | 分层可导航小世界图。用于快速近似最近邻搜索的多层图 |
| L1 正则化 | "Lasso" | 将权重的 L1 范数加到损失中。将权重推向零（稀疏性） |
| L2 正则化 | "Ridge"或"权重衰减" | 将权重的 L2 范数平方加到损失中。将权重向零收缩而不产生稀疏性 |
| 弹性网络 (Elastic Net) | "L1 + L2" | 结合 L1 和 L2 正则化。处理相关特征组时优于单独使用任一方法 |
