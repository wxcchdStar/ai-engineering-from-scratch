# 失败模式——MAST、群体思维、单一文化、级联错误

> 2026 年的参考分类法是 **MAST**（Cemri 等人，NeurIPS 2025，arXiv:2503.13657），源自 7 个最先进开源 MAS 的 1642 条执行轨迹，显示 **41-86.7% 的失败率**。三个根类别：**规范问题**（41.77%）——角色模糊、任务定义不清晰；**协调失败**（36.94%）——通信中断、状态不同步；**验证缺口**（21.30%）——缺少验证、缺少质量检查。**群体思维（Groupthink）**家族（arXiv:2508.05687）增加了：单一文化崩溃（相同基础模型 → 相关失败）、从众偏差（智能体相互强化错误）、有缺陷的心智理论、混合动机动态、级联可靠性失败。级联示例：重试风暴——支付失败触发订单重试，订单重试触发库存重试，库存重试压垮库存服务（数秒内 10 倍负载——需要断路器）。内存中毒：一个智能体的幻觉进入共享内存，下游智能体将其视为事实；准确率逐渐衰减，使根因诊断痛苦。**STRATUS**（NeurIPS 2025）报告通过专门的检测 / 诊断 / 验证智能体实现了 1.5 倍的缓解成功率提升。本课将失败模式视为一等工程目标。

**类型：** 学习
**语言：** Python（标准库）
**前置要求：** 第 16 阶段 · 13（共享内存），第 16 阶段 · 14（共识与拜占庭容错），第 16 阶段 · 15（投票与辩论拓扑）
**时间：** 约 75 分钟

## 问题

多智能体系统在实际任务上失败的概率为 41-86.7%（Cemri 等人 2025 年在 7 个开源 MAS 上测量）。这不是"再加更多智能体"就能调试的。失败有结构性原因。MAST 分类法给你类别。本课将每个类别映射到具体的检测、诊断和缓解模式，使数字不再看起来随意。

2026 年的生产实践是将失败模式视为设计输入。你的架构在你能指出每个 MAST 类别并说出你部署的缓解措施之前，不算"足够好"。

## 概念

### MAST 类别

**规范问题（占失败的 41.77%）。** 智能体的任务定义不够严格。示例：

- 角色模糊：两个智能体都认为自己是审查者。
- 任务规格不足："总结这个"，而用户想要特定角度。
- 成功标准隐含：智能体无法判断是否成功。

缓解措施：
- 编写显式的角色契约。每个智能体的提示说明它做什么*和不做什么*。
- 每个任务的验收测试。在智能体开始之前，定义"完成的样子是 X"。
- 派发前规范检查：一个单独的智能体在派发前审查任务定义。

**协调失败（36.94%）。** 通信或状态故障。

示例：
- 两个智能体在没有同步的情况下更新共享状态。
- 智能体之间的消息丢失（队列故障、超时）。
- 状态漂移：智能体 A 认为任务完成；智能体 B 仍在执行。

缓解措施：
- 带乐观并发的版本化共享状态。
- 对关键消息的显式确认（重试直到确认）。
- 定期状态同步检查点；及早检测漂移。

**验证缺口（21.30%）。** 对输出没有独立检查。

示例：
- 一个智能体声称成功；没有人验证。
- 智能体链中每个都信任前一个的输出。
- 对涌现的组合行为缺少测试覆盖。

缓解措施：
- 独立验证智能体（第 13 课）。只读，独立源访问。
- 显式交接契约："A 的输出必须通过检查器 C 的验证，B 才能开始。"
- 用于事后分析的结果日志。

### 群体思维家族（arXiv:2508.05687）

当智能体同质化或相互模仿时的五种相关失败：

**单一文化崩溃。** 相同的基础模型或训练数据 → 相关错误。当三个智能体共享一个 LLM 时，它们共享其幻觉。

**从众偏差。** 智能体向最大声或最自信的同伴调整，即使它是错的。

**有缺陷的心智理论（ToM）。** 智能体无法建模彼此的信念；协调崩溃（第 18 课）。

**混合动机动态。** 具有部分对齐激励的智能体漂向折中中间地带，这不能满足任何人。

**级联可靠性失败。** 一个组件的错误模式触发依赖组件中的错误模式。

### 级联示例——重试风暴

一个经典的 2026 年事件模式：

```
支付服务 10% 的请求失败
   ↓
订单智能体重试支付（指数退避但天真）
   ↓
每次重试都是一次新的订单-库存检查
   ↓
库存服务看到 2 倍正常负载
   ↓
库存服务开始超时
   ↓
每个订单重试库存检查
   ↓
库存服务看到 10 倍正常负载
   ↓
集群宕机
```

修复是经典的：**断路器**。当下游错误率超过阈值时，用缓存或默认结果短路。加上每次请求的上限重试预算。

断路器是少数几个你直接从分布式系统借用而无需修改的多智能体失败缓解措施之一。

### 内存中毒（重温）

来自第 13 课：一个智能体的幻觉成为共享内存事实；下游智能体在被污染的事实上推理。用 MAST 术语，这是共享内存层的验证缺口。

逐渐准确率衰减是症状。你不会得到崩溃；你会得到难以根因的缓慢漂移。

缓解：仅追加日志、来源追踪、不可写验证者。已在第 13 课中涵盖。

### STRATUS——用于失败检测的专门智能体

STRATUS（NeurIPS 2025）报告当你部署以下内容时，缓解成功率提升 1.5 倍：

- **检测智能体。** 监视症状模式（高分歧、重试激增、准确率漂移）。
- **诊断智能体。** 给定症状，从 MAST 分类法推断可能的根因。
- **验证智能体。** 在应用缓解措施后，检查症状是否清除。

这是 SRE 风格的事件响应，应用于智能体系统。三个角色都可以是具有专门提示的 LLM 智能体。

### 失败模式审计

2026 年的最佳实践是每年（或每个主要版本）的失败模式审计：

1. **轨迹采样。** 收集约 1000 条真实执行轨迹。
2. **分类。** 对于每条轨迹的失败，映射到 MAST + 群体思维类别。
3. **计算按类别的失败率。** 哪些类别主导你的系统？
4. **排名缓解措施。** 哪个修复会消除最多的失败？
5. **选择 2-3 个缓解措施。** 实施；下个季度重新审计。

纪律比具体选择更重要。没有审计，失败会融入噪音，永远不会被系统性地解决。

### 当系统静默失败时

最危险的失败类别是静默正确性失败。一个大声失败的系统（崩溃、异常、告警）可以被监控。一个产生看似合理但错误输出的系统无法通过异常日志检测。这就是为什么验证缺口是每次失败最昂贵的类别，尽管按数量只占 21.30%。

投资于：
- 基于采样的人工审查。
- 黄金数据集回归测试。
- 对重要输出的跨智能体交叉检查。

### 失败 vs 缓慢失败

有些失败是即时的；有些是缓慢的。即时失败（超时、模式不匹配、认证错误）检测成本低。缓慢失败（内存中毒、单一文化漂移、角色模糊）检测和预防成本高。

2026 年的工程举措：对缓慢失败代理进行仪表化，以便在漂移变成可见错误之前捕获它。一致率、重试率、输出长度分布以及连续智能体版本之间的编辑距离都是有用的代理指标。

`code/main.py` 实现了：

- 检测智能体标记模式并命名 MAST 类别。

`outputs/skill-mast-auditor.md` 对多智能体系统运行 MAST 风格的失败模式审计。轨迹 → 分类 → 缓解排名。

## 交付它

生产中的失败模式纪律：

- **每季度 MAST 审计。** 不是每年。类别随着系统增长而变化。
- **处处断路器。** 对任何依赖服务的每个出站调用。默认打开阈值在 5-10% 错误率。
- **黄金数据集。** 小型、高质量、人工审计。每周对其进行回归测试。
- **STRATUS 三重奏。** 监控生产的检测 + 诊断 + 验证智能体。仅从检测智能体开始；当症状嘈杂时添加诊断。
- **失败预算。** 按类别的失败率显式 SLO。超出预算触发停止发布的对话。

## 练习

1. 运行 `code/main.py`。确认断路器限制了重试风暴。改变失败阈值并观察权衡。
2. 实现一个**缓慢失败代理**：3 个并行智能体之间的一致率。当它急剧下降时，触发告警。通过逐渐关联智能体输出来模拟单一文化漂移。
3. 阅读 Cemri 等人（arXiv:2503.13657）。选择他们 7 个 MAS 系统中的一个，映射其前 3 个失败类别。这些与 MAST 预测的相比如何？
4. 阅读群体思维论文（arXiv:2508.05687）。确定五种模式中哪种在生产中最难检测。提出一个代理指标。
5. 为你了解的一个特定多智能体系统设计一个 STRATUS 风格的检测-诊断-验证三重奏。检测监视哪些症状？诊断推荐哪些缓解措施？验证如何确认它们有效？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|----------|
| MAST | "2026 年分类法" | Cemri 2025；3 个根类别 + 14 个失败子类型。 |
| 规范问题 | "角色模糊" | 任务或角色定义不足；智能体不知道该做什么。 |
| 协调失败 | "状态漂移" | 智能体之间的通信或同步故障。 |
| 验证缺口 | "没有人检查" | 输出在没有独立验证的情况下被接受。 |
| 群体思维家族 | "同质化失败" | 单一文化、从众、有缺陷的 ToM、混合动机、级联。 |
| 单一文化崩溃 | "相同模型，相同幻觉" | 来自共享基础模型或训练数据的相关错误。 |
| 重试风暴 | "级联错误放大" | 一个失败触发重试，重试放大下游负载。 |
| 断路器 | "错误率过高时快速失败" | 当错误率超过阈值时打开；用默认值短路。 |
| STRATUS | "事件响应三重奏" | 检测 + 诊断 + 验证智能体。1.5 倍缓解成功。 |
| 内存中毒 | "幻觉传播" | 共享内存事实被污染；下游智能体在被污染事实上推理。 |
