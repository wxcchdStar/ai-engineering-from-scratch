# 共识与拜占庭容错（Consensus and Byzantine Fault Tolerance for Agents）

> 经典分布式系统 BFT 遇上随机 LLM。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探测（confidence probe）加权每票；**DecentLLMs**（arXiv:2507.14928）采用无领导者并行工作者提案和几何中位数（geometric-median）聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类（Hierarchical Structure Clustering）相结合，以拆分核心（Core）和边缘（Edge）节点。"Can AI Agents Agree?"（arXiv:2603.01213）的诚实实证结果是，即使是标量一致性在今天也是脆弱的——单个欺骗性智能体可以破坏混合智能体（Mixture-of-Agents）系统。BFT 是必要的但不充分。本课程构建一个最小化 BFT 协议，注入三种智能体特定攻击（拜占庭谎言、谄媚从众、相关误差单一文化），并测量每种共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 16 阶段 · 07（心智社会与辩论）、第 16 阶段 · 13（共享内存）
**时间：** ~75 分钟

## 问题

你有 N 个 LLM 智能体，每个产生一个答案。它们意见不一致。多数投票选择了错误的一个，因为两个智能体是相关的（相同的基础模型、相同的训练数据、相同的故障模式）。第三个智能体恰好以一种新颖的方式出错——因此多数是虚假多数。

现在添加一个欺骗性智能体：它故意撒谎。或者一个谄媚智能体：它同意最后发言的人。在经典 BFT 中，假设拜占庭节点是 `f < n/3` 的一部分并且行为任意。2026 年的现实是，LLM 节点即使在诚实的情况下也是随机的，跨模型相关，并受彼此输出的影响。你不能将它们视为独立的伯努利投票者。

经典 BFT（PBFT，1999）不是错误的——它是不完整的。它处理任意位翻转。它不处理"三个诚实智能体共享幻觉，因为它们共享训练数据"。本课程从 PBFT 的基础构建，并叠加三种 2025-2026 年的改编。

## 概念

### 经典 BFT 给你什么

实用拜占庭容错（Practical Byzantine Fault Tolerance, Castro & Liskov, OSDI 1999）容忍 `f < n/3` 个拜占庭节点。协议有三个阶段（pre-prepare、prepare、commit）和两个原语（签名消息、法定人数证书）。在 `n >= 3f + 1` 个诚实或恶意节点中就单个值达成一致。

保证很强，但假设：

1.  **独立故障。** 拜占庭节点不协调。
2.  **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3.  **问题有真实答案。** 对错误事实的共识仍然是共识。

LLM 智能体违反所有三条。两个运行相同基础模型的智能体共享故障。"诚实"的 LLM 仍然会产生幻觉。在模糊的问题上，"真相"是智能体决定的——没有外部预言机。

### 三种 LLM 特定攻击

**拜占庭谎言。** 一个智能体输出故意错误的答案。如果 `f < n/3`，经典 BFT 处理此问题。

**谄媚从众。** 一个智能体在投票前读取他人的答案，并与最后发言的人保持一致。不是恶意的，但与最响亮的声音相关。经典 BFT 不能防止这一点，因为智能体通过了每个签名检查。

**相关误差单一文化。** 三个智能体共享一个基础模型。它们产生幻觉相同的错误答案。多数是错误的。经典 BFT 没有帮助，因为所有三个都"诚实地"同意。

### 2025-2026 年的回应

**CP-WBFT**（arXiv:2511.10400）——置信度探测加权 BFT。每个投票者将置信度探测附加到其答案（自报概率，或单独校准模型的预测）。投票权重随置信度缩放。报告在完全图上 BFT 改进 +85.71%。缓解：谄媚从众（从众智能体在其自愿立场上往往置信度较低）。

**DecentLLMs**（arXiv:2507.14928）——无领导者。工作者智能体并行提案，评估者智能体对提案评分，最终答案是评分位置的几何中位数。当 `f < n/2` 时鲁棒。缓解：拜占庭谎言和相关误差（几何中位数对异常值鲁棒，并拉向密集簇，而不是模型偏差的平均值）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史中学习的信任分数分配。将智能体聚类为核心（Core）和边缘（Edge）；核心智能体必须首先达成共识，边缘智能体跟随。缓解：可扩展性（核心共识小而快）和部分单一文化（核心可以选择多样性）。

### 实证："Can AI Agents Agree?"（arXiv:2603.01213）

该论文测量了跨多个前沿模型的标量一致性（LLM 智能体就单个数值达成一致）。发现令人不安：

-   即使没有对手，LLM 智能体在许多基准测试上对标量问题的分歧率超过 30%。
-   单个采用欺骗性角色的智能体可以将混合智能体共识从诚实基线拉偏 40+ 个百分点。
-   分歧率与模型多样性相关——异构集成比同构集成分歧更多（好：不相关误差），但也漂移更慢（坏：达成一致的时间更长）。

要点：BFT 为你提供了对齐输出的机制，但它不告诉你对齐的输出是否正确。结合验证（第 16 阶段 · 08 角色专业化）、多样性（第 16 阶段 · 15 辩论变体）和评估者智能体（第 16 阶段 · 24 基准测试）。

### 核心协议，精简版

LLM 智能体的最小化 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探测 c_i ∈ [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义簇（等价答案）分组
5. 聚合器计算每个簇 C 的权重：
     w(C) = sum_{i in C} c_i
6. 获胜者 = 具有最大权重的簇，如果 max > threshold * sum(c_i)
   否则：重试或升级
7. 少数簇记录溯源以供事后审计
```

语义聚类步骤是 LLM 特定的转折。两个答案"研究报告 4.2%"和"4.2% 提升"是同一个簇。天真的字符串相等检查会错过这一点。在生产中，使用廉价的嵌入模型或显式规范化。

### 阈值调优

`threshold` 参数决定何时接受和何时重试。太低：你接受弱多数。太高：你从不接受任何东西。经验范围：`n=5-7` 个智能体为 0.5-0.67，较小的 `n` 更高。低于阈值，升级到人类或不同的智能体集成。

### 共识在哪里没有帮助

-   **模糊问题。** 如果问题没有真实答案，共识是一种意见。就这样称呼它。
-   **复合问题。** "编写代码并解释它"——两个答案。分别对每个投票。
-   **对抗性多轮。** 如果智能体可以观察先前的轮次并模仿（Du 2023 辩论），它们开始相互同意，无论真相如何。限制轮次（通常 2-3 轮）。

## 构建它

`code/main.py` 实现：

-   `AgentVoter`——带有（答案，置信度）的脚本化策略。
-   `MajorityVote`——经典多数制。
-   `CPWBFT`——带语义聚类的置信度加权投票。
-   `DecentLLMs`——对评分提案的几何中位数聚合。
-   `Scenario`——在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1.  `byzantine`：一个智能体以高置信度撒谎。
2.  `sycophancy`：一个智能体复制它看到的第一个答案，具有匹配的置信度。
3.  `monoculture`：三个智能体共享一个错误答案（相关误差），具有中等置信度。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）→ 最终答案的表格，正确答案高亮显示。多数制在单一文化案例中失败。CPWBFT 的置信度加权缓解了谄媚。当单一文化少于人口一半时，DecentLLMs 的几何中位数拉向诚实簇。

## 交付它

在交付任何共识机制之前：

-   **至少用上述三种模式进行攻击测试。** 你的协议应可预测地失败，而不是静默地失败。
-   **记录每个少数簇**及其溯源。少数簇是你相关误差的早期预警系统。
-   **强制有界轮次。** 不要"一直辩论直到达成一致"——这会奖励谄媚。
-   **将一致性与正确性分开。** 共识输出交给验证器；验证器独立于集成。
-   **监控一致性率。** 急剧上升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1.  运行 `code/main.py`。确认多数制在单一文化攻击中失败，但当单一文化置信度低于 0.7 时 CPWBFT 部分缓解了它。
2.  添加第四种攻击模式：**静默弃权**——一个智能体拒绝回答（"我不知道"）。每个聚合器应如何处理弃权？实现你的选择。
3.  将语义聚类从字符串规范化切换为嵌入相似度（使用任何开源嵌入模型）。谄媚攻击会发生什么？
4.  阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（单独的校准模型检查每个智能体的自报置信度）。测量单一文化场景上的准确性增益。
5.  阅读"Can AI Agents Agree?"（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个智能体，一个标量问题，欺骗性角色提示词。CPWBFT 或 DecentLLMs 能捕捉到它吗？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|----------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，用于 `f < n/3` 任意故障的共识。 |
| 拜占庭 | "任何不良行为" | 一个可以撒谎、丢弃消息、静默失败的节点——除了安全崩溃之外的任何行为。 |
| 置信度探测 | "你有多确定？" | 附加到投票的自报或校准器预测的概率。 |
| 语义聚类 | "相同答案，不同措辞" | 在计算票数之前对等价答案进行分组。 |
| 几何中位数 | "鲁棒中心" | 最小化到样本点距离和的点。对异常值鲁棒，不同于均值。 |
| 单一文化 | "相同模型，相同故障" | 当智能体共享训练数据或基础模型时的相关误差。 |
| 谄媚从众 | "同意响亮的声音" | 智能体的投票偏向于最先/最响亮发言的人。 |
| 核心/边缘 | "层次 BFT" | WBFT 拆分：小核心先达成共识，边缘节点跟随。限制延迟。 |
