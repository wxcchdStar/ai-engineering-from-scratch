# MARL — MADDPG、QMIX、MAPPO

> 多智能体协调的强化学习遗产，在 2026 年仍然影响着 LLM 智能体系统。**MADDPG**（Lowe 等人，NeurIPS 2017，arXiv:1706.02275）引入了集中式训练、分散式执行（Centralized Training, Decentralized Execution，CTDE）：每个评论家（critic）在训练期间看到所有智能体的状态和动作；在测试时只有局部的演员（actor）运行。适用于合作、竞争和混合场景。**QMIX**（Rashid 等人，ICML 2018，arXiv:1803.11485）是带有单调混合网络的值分解（value-decomposition）；每个智能体的 Q 值组合成联合 Q 值，使得 `argmax` 可以干净地分布——在星际争霸多智能体挑战赛（StarCraft Multi-Agent Challenge，SMAC）上占据主导地位。**MAPPO**（Yu 等人，NeurIPS 2022，arXiv:2103.01955）是带有集中式值函数的 PPO；在粒子世界（particle-world）、SMAC、Google Research Football、Hanabi 上"出人意料地有效"，且只需极少的调参。这些算法支撑着必须分散行动的智能体团队的训练策略。MAPPO 是**2026 年默认的合作式 MARL 基线**。本课在一个小型网格世界玩具上构建每种算法，在接触 LLM 智能体训练之前将这三个思想植入肌肉记忆。

**类型：** 学习
**语言：** Python（标准库，不依赖 NumPy 的小型实现）
**前置要求：** 第 09 阶段（强化学习），第 16 阶段 · 09（并行/群体/网络架构）
**时间：** 约 90 分钟

## 问题

LLM 智能体系统越来越多地训练智能体间协调的策略：何时推迟、何时行动、调用哪个同伴。告诉你如何训练此类策略的文献是多智能体强化学习（Multi-Agent Reinforcement Learning，MARL），它早于 LLM 浪潮，并拥有一小组主导算法。

在没有模式词汇表的情况下阅读 MARL 论文是痛苦的。集中式训练与分散式执行（CTDE）、值分解和集中式评论家不是流行语——它们是对特定问题的特定答案：

- 独立 RL（每个智能体单独学习）从每个智能体的角度看是非平稳的（non-stationary）。不好。
- 集中式 RL（一个智能体控制所有）无法扩展且违反执行约束。
- CTDE 兼具两者之长：用全局信息训练，用局部策略部署。

## 概念

### 论文使用的三种环境

- **粒子世界（Multi-Agent Particle Env）。** 简单的 2D 物理，包含合作/竞争任务。MADDPG 的原始测试平台。
- **星际争霸多智能体挑战赛（SMAC）。** 合作式微操，部分可观测。QMIX 的测试平台。离散动作，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO 的基线。

不同环境有不同的动作/观测类型。算法相应地选择。

### MADDPG（2017）——CTDE 模式

每个智能体 `i` 有一个演员 `mu_i(o_i)`，将其自身的观测映射为动作。每个智能体还有一个评论家 `Q_i(x, a_1, ..., a_n)`，在训练期间看到所有观测和所有动作。演员通过针对评论家评估的策略梯度进行更新。

```
演员更新：    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) 在 a_i=mu_i(o_i) 处]
评论家更新：  对 Q_i(x, a_1..n) 进行 TD 更新，给定下一状态的联合估计
```

为什么需要 CTDE：在训练时，我们知道每个人的动作；我们利用这一点来降低每个评论家的方差。在部署时，每个智能体只看到 `o_i` 并调用 `mu_i(o_i)`。

失败模式：评论家随 N 个智能体增长（输入包含所有动作）。在没有近似的情况下无法扩展到约 10 个以上智能体。

### QMIX（2018）——值分解

仅限合作。全局奖励是每个智能体 Q 值的单调函数之和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证了 `argmax_a Q_tot` 可以通过每个智能体独立选择 `argmax_{a_i} Q_i` 来计算。这正是你需要的**分散式执行属性**。在训练时，一个混合网络从每个智能体的 Q 值中产生 `Q_tot`。

为什么 QMIX 在 SMAC 上获胜：合作式星际争霸微操具有同质智能体、局部观测、全局奖励——完美适合值分解。

失败模式：单调性约束是限制性的；某些任务的奖励结构不是单调可分解的（一个智能体为团队牺牲）。扩展（QTRAN、QPLEX）放宽了这一点。

### MAPPO（2022）——被忽视的默认选择

多智能体 PPO：带有集中式值函数的 PPO。每个智能体有自己的策略；所有智能体共享（或拥有各自的）值函数，这些值函数看到完整状态。Yu 等人 2022 年在五个基准上将 MAPPO 与 MADDPG、QMIX 及其扩展进行了对比，发现：

- MAPPO 在粒子世界、SMAC、Google Research Football、Hanabi、MPE 上匹敌或击败离策略（off-policy）MARL 方法。
- 需要极少的超参数调优。
- 训练稳定；跨随机种子可复现。

社区在这篇论文之前低估了同策略（on-policy）MARL。在 2026 年，MAPPO 是合作式 MARL 的默认基线；任何新方法都必须击败它。

### 为什么 LLM 智能体工程师应该关心

三个直接用途：

1. **路由器训练。** 一个元智能体选择哪个子智能体处理任务。这是一个 MARL 问题，有 N 个分散的子智能体和一个集中式路由器。MAPPO 适用。
2. **角色涌现。** 在生成式智能体模拟中，训练智能体随时间采用互补角色本质上是一个 MARL 问题。QMIX 风格的值分解通过构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过 CTDE 训练它们会产生尊重资源约束的可部署局部策略。

实际注意事项：在 2026 年，大多数生产级 LLM 智能体系统通过提示（prompt）而非训练来制定策略。MARL 在你有（a）大量交互数据，（b）明确的奖励信号，以及（c）愿意投资训练基础设施时才会介入。

### CTDE 作为超越 RL 的设计模式

即使没有训练，CTDE 也是一种有用的架构模式：

- 在*设计*期间，假设完整的团队可见性。
- 在*运行时*，强制执行分散式执行：每个智能体只看到 `o_i`。

该模式迫使你将每个智能体的状态保持显式，并提前考虑部分可观测性。许多生产级多智能体系统默默地假设处处共享状态——CTDE 纪律可以防止这种情况。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（包括其他智能体的策略）是非平稳的。经典的单智能体 RL 证明失效。本课中的所有 MARL 算法都解决了这个问题：

- MADDPG：全局评论家看到所有动作，因此其值估计是平稳的。
- QMIX：值分解将学习移到一个联合 Q 空间，其中最优性是明确定义的。
- MAPPO：集中式值函数抑制了来自其他智能体策略变化的方差。

在 LLM 智能体系统中，非平稳性表现为"我的智能体上个月还能用，现在上游的另一个智能体变了，我的就出问题了。"用 CTDE 训练 MARL 是原则性的修复；提示级别的修复更快但不够持久。

### 本课不涵盖的内容

训练实际网络是第 09 阶段的话题。本课构建脚本化策略版本，演示 CTDE、值分解和集中式值模式，无需梯度更新。目标是在你拿起完整的 MARL 库（PyMARL、MARLlib、RLlib 多智能体）之前内化这些模式。

## 构建它

`code/main.py` 实现了三个模式演示，全部在一个微小的 2 智能体合作网格世界上：

- 环境：2 个智能体在 4x4 网格上，一个奖励球。奖励 = 1（如果任何智能体到达球）；任务完成。
- `IndependentAgents`——每个智能体将其他智能体视为环境。基线。
- `MADDPGStyle`——集中式评论家计算联合值；演员策略从中更新。脚本化策略改进。
- `QMIXStyle`——带有单调混合器的值分解。
- `MAPPOStyle`——集中式值函数；策略针对共享基线进行更新。

所有四种方法运行相同的回合并报告平均到达目标的步数。CTDE 变体比独立基线收敛到更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约 6 步；CTDE 变体收敛到约 3.5 步（4x4 网格的最优值是 3）。尽管是脚本化策略，模式差异仍然显现。

## 使用它

`outputs/skill-marl-picker.md` 是一个技能，为给定的多智能体任务选择 MARL 算法：合作 vs 竞争、同质 vs 异质、动作空间类型、规模、奖励信号。

## 交付它

MARL 在生产中很少见。当你确实使用时：

- **从 MAPPO 开始。** 2022 年的论文将其确立为基线；先复现它可以节省数周追逐更花哨方法的时间。
- **记录每个智能体的观测和动作流。** 没有每个智能体的轨迹，调试 MARL 是无望的。
- **将训练代码与执行代码分离。** CTDE 是一种纪律；让执行路径真的只看到 `o_i`。
- **奖励塑形警告。** MARL 对奖励设计极其敏感。塑形中的一个协调 bug，智能体就会学会利用它。运行对抗性测试。
- **对于 LLM 智能体**，首先考虑提示级别的策略。只有在交互数据 + 奖励信号 + 基础设施全部到位时，才投资 MARL 训练。

## 练习

1. 运行 `code/main.py`。测量独立智能体和 MAPPO 风格智能体之间的到达目标步数差距。在 6x6 网格上，这个差距是扩大还是缩小？
2. 实现一个竞争变体：两个智能体，一个球，只有第一个到达的获得奖励。哪种模式能干净地处理竞争？历史上是 MADDPG。
3. 阅读 MADDPG（arXiv:1706.02275）第 3 节。用你自己的话以伪代码符号化地实现精确的评论家更新规则。
4. 阅读 MAPPO（arXiv:2103.01955）。为什么作者认为集中式值 + PPO 在他们的基准上击败了离策略 MARL？列出三个最强的主张。
5. 将 CTDE 作为设计模式应用到一个假设的 LLM 智能体系统（例如，研究智能体 + 摘要器 + 编码器）。在设计时可用的联合信息中，哪些在运行时不可用？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|----------|
| MARL | "多智能体 RL" | 多智能体系统的强化学习。 |
| CTDE | "集中式训练，分散式执行" | 用全局信息训练；用局部策略部署。 |
| MADDPG | "多智能体 DDPG" | 每个智能体的评论家看到所有观测 + 动作的 CTDE。 |
| QMIX | "值分解" | 每个智能体 Q 值的单调混合。合作式。 |
| MAPPO | "多智能体 PPO" | 带有集中式值函数的 PPO。2026 年默认基线。 |
| 值分解 | "个体 Q 值之和" | 联合 Q 表示为每个智能体 Q 值的单调函数。 |
| 非平稳性 | "移动的目标" | 每个智能体的环境随着其他智能体的学习而变化。核心 MARL 问题。 |
| 同策略 / 离策略 | "从当前 / 回放中学习" | PPO 是同策略（MAPPO）；DDPG 和 Q-learning 是离策略。 |
| SMAC | "星际争霸多智能体挑战赛" | 合作式微操基准；QMIX 的主场。 |
