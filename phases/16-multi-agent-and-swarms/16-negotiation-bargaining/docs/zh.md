# 谈判与讨价还价（Negotiation and Bargaining）

> 智能体就资源、价格、任务分配和条款进行谈判。2026 年的基准测试集很清晰：NegotiationArena（arXiv:2402.05863）显示 LLM 可以通过角色操纵（"绝望"）将收益提高约 20%；"Measuring Bargaining Abilities"（arXiv:2402.15813）显示买方比卖方更难，规模没有帮助——他们的 **OG-Narrator**（确定性报价生成器 + LLM 叙述者）将成交率从 26.67% 提高到 88.88%；大规模自主谈判竞赛（Large-Scale Autonomous Negotiation Competition, arXiv:2503.06416）运行了约 18 万次谈判，发现**隐藏思维链（chain-of-thought-concealing）**的智能体通过对对手隐藏推理而获胜；Bhattacharya 等人 2025 年关于哈佛谈判项目（Harvard Negotiation Project）指标的研究将 Llama-3 评为最有效，Claude-3 最具攻击性，GPT-4 最公平。本课程实现 Contract Net Protocol（合同网协议，FIPA 祖先，第 02 课），连接 LLM 风格的买方/卖方，运行 OG-Narrator 风格的分解，并测量每种结构性选择如何改变成交率。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 16 阶段 · 02（FIPA/ACL 传统）、第 16 阶段 · 09（并行群体网络）
**时间：** ~75 分钟

## 问题

两个智能体需要就价格达成一致。在纯语言提示词下，2024-2026 年的 LLM 以令人惊讶的低成交率达成交易（在 arXiv:2402.15813 的严格参数化讨价还价中约为 27%）。规模不能解决这个问题：GPT-4 在讨价还价方面并不比 GPT-3.5 结构上更好；它在讨价还价的*语言*方面更好。

根本问题是 LLM 混淆了两个工作——决定报价和叙述报价。OG-Narrator 将它们分开：确定性报价生成器计算数值移动；LLM 只叙述。成交率跃升至约 89%。

这反映了一个经典的多智能体发现：将机制与通信层解耦会胜出。Contract Net Protocol（FIPA, 1996; Smith, 1980）是参考任务市场机制。将 LLM 插入叙述槽，你就得到了一个现代的 LLM 驱动的任务市场。

## 概念

### Contract Net，一段话

Smith 1980 年的 Contract Net Protocol：**管理者（manager）**广播**招标（call for proposals, cfp）**；**竞标者（bidders）**用包含其报价的 **propose** 消息回应；管理者选择获胜者并向获胜者发送 **accept-proposal**，向失败者发送 **reject-proposal**。获胜者执行工作。可选消息：**refuse**（竞标者拒绝提案）。FIPA 将其编撰为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 胜出

"Measuring Bargaining Abilities of Language Models"（arXiv:2402.15813）观察到：

-   LLM 经常违反讨价还价规则（以无意义的价格报价，忽略对方的 ZOPA）。
-   它们锚定不佳（接受糟糕的首次报价；以象征性而非战略性金额还价）。
-   仅靠规模不能解决这些问题。更大的模型产生更合理的语言，但战略错误相似。

OG-Narrator 分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  状态  →  │   报价生成器     │ 价格 → │   LLM 叙述者     │ → 消息
           │  （确定性的）     │        │  （编写人性化     │
           │                  │        │   伴随文本）      │
           └──────────────────┘        └──────────────────┘
```

报价生成器是经典谈判策略：Rubinstein 讨价还价模型、Zeuthen 策略或简单的价格 tit-for-tat。LLM 叙述。消息包含确定性价格和自然语言框架。

成交率跃升因为：
-   价格保持在讨价还价区域内。
-   锚点是战略性的，不是情绪化的。
-   LLM 做它擅长的事：写作。

### NegotiationArena 发现

arXiv:2402.05863 提供了规范基准测试。头条发现：

-   LLM 可以通过采用角色（"我绝望地想在周五之前卖掉这个"）将收益提高约 20%——角色操纵是一种真实的策略。
-   公平/合作的智能体被对抗性智能体利用；防御需要显式的反姿态。
-   对称配对在大约 40% 的基准测试场景中收敛到不公平的结果。

这不是"LLM 是糟糕的谈判者"。而是"LLM 谈判太像人类了，包括可被利用的部分"。

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）在多种 LLM 策略上运行了约 18 万次谈判。获胜者对对手隐藏其推理：

-   如果智能体将"我只会出到 $75；我的保留价格是 $70"打印到公开可见的草稿板上，对手会读取它。
-   获胜者私下计算策略；输出通道只包含报价和最低要求的叙述。

这是 2026 年对经典博弈论的回响（Aumann 1976 关于理性和信息）：透露你的私人估值会损失收益。LLM 不会直觉到这一点，并愉快地将保留价格输入到对对手可见的推理跟踪中。

工程要点：将私人草稿板上下文与公开消息上下文分开。不是可选的。

### Bhattacharya 等人 2025——模型排名

在哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）上：

-   **Llama-3**在达成交易方面最有效（成交率 + 收益）。
-   **Claude-3**是最具攻击性的谈判者（高锚定、晚期让步）。
-   **GPT-4**是最公平的（跨配对收益方差最小）。

这是 2025 年的快照。重点不是 2026 年 4 月哪个模型获胜——而是不同的基础模型具有持久的谈判风格。异构集成（第 15 课）将其作为多样性来源。

### 通过 Contract Net + LLM 进行任务分配

Contract Net 对 LLM 多智能体的现代重用：

1.  管理者智能体将任务分解为单元。
2.  向工作者智能体广播带有任务描述的 `cfp`。
3.  每个工作者返回报价：`(price, eta, confidence)`，其中价格可以是 token、计算单元或美元。
4.  管理者选择获胜者（单个或多个，取决于任务）并授予。
5.  被拒绝的工作者可以自由竞标其他任务。

这可以很好地扩展到 100 个以上的工作者，因为协调是广播和响应，而不是同步聊天。用于生产：Microsoft Agent Framework 的编排模式，一些 LangGraph 实现。

### LLM 利益相关者交互式谈判

NeurIPS 2024 引入了带有**秘密分数**和**最低接受阈值**的多方可评分博弈。每个利益相关者都有私人效用；LLM 必须从消息中推断它们。这是两方讨价还价到 N 方联盟形成的推广。与具有异构工作者能力的生产任务市场相关。

### 叙述 vs 机制规则

在所有 2024-2026 年谈判基准测试中，一致的工程规则是：

> 让 LLM 叙述。不要让 LLM 计算报价。

如果报价需要是一个数字（价格、ETA、数量），从谈判状态确定性地生成它，并让 LLM 生成框架。如果报价需要是一个提案结构（任务分解、角色分配），让 LLM 起草它，但在发送之前根据模式和约束检查进行验证。

## 构建它

`code/main.py` 实现：

-   `ContractNetManager`、`ContractNetTask`、`Bid`——管理者 + 竞标者，广播 cfp，收集提案，授予。
-   `og_narrator_bargain(state, rng)`——OG-Narrator 买方：确定性 Zeuthen 风格向中点让步。
-   `seller_response(state, rng)`——确定性卖方还价策略（两种风格的结构性真实基准）。
-   `naive_llm_bargain(state, rng)`——模拟全 LLM 讨价还价者：以高方差选择价格，通常超出 ZOPA。
-   测量：1000 次试验的成交率，每次试验采样新的保留价格。

运行：

```
python3 code/main.py
```

预期输出：天真 LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 个百分点的差距是将报价生成与叙述分解的结构性优势。加上一个带有三个竞标者和一个任务的 Contract Net 任务市场分配示例。

## 交付它

生产级讨价还价清单：

-   **分离草稿板。** 私人状态永远不会到达对手的上下文。这是不可协商的。
-   **确定性报价生成。** 价格、数量、ETA：计算，不要提示。
-   **根据模式验证所有传入报价。** 在协议边界拒绝超出 ZOPA 的报价。
-   **有界轮次。** 最多 3-5 轮；在僵局时升级到调解者。
-   **持续测量成交率和收益方差。** 成交率下降是一个症状——通常是提示词漂移或对手侧攻击。
-   **记录所有被拒绝的提案**及其确定性理由。对于 Contract Net 管理者，失败的竞标者需要理解原因。

## 练习

1.  运行 `code/main.py`。确认 OG-Narrator 在成交率上击败天真 LLM。差距多大？
2.  实现**基于角色的收益提升**（arXiv:2402.05863）——买方仅在叙述中采用"本周绝望购买"的角色，报价生成器不变。成交率或收益会改变吗？
3.  实现思维链**隐藏**：维护一个不传递给对手的私人草稿板字符串。如果你意外泄露它（通过交换通道模拟），会发生什么？
4.  将 Contract Net 扩展为带有保留价格的 N 竞标者拍卖。当所有出价都超过保留价时，管理者如何在最低价格和最高质量之间决定？你选择哪种授予规则，为什么？
5.  阅读 Bhattacharya 等人 2025 关于哈佛谈判项目指标的研究。实现两个具有不同风格（攻击性 vs 公平）的讨价还价者。测量对称和不对称配对下的收益方差。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|----------|
| Contract Net | "任务市场" | Smith 1980, FIPA 1996。cfp + propose + accept/reject。规范任务市场。 |
| ZOPA | "可能达成协议的区域" | 买方最高价和卖方最低价之间的重叠。超出此范围的报价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果此交易失败，你的后备方案。设定你的保留价格。 |
| OG-Narrator | "报价生成器 + 叙述者" | 分解：确定性报价，LLM 叙述。 |
| Zeuthen 策略 | "风险最小化让步" | 经典报价生成器，基于风险限制让步。 |
| Rubinstein 讨价还价 | "交替报价均衡" | 带有折现的无限期讨价还价的博弈论模型。 |
| CoT 隐藏 | "隐藏你的推理" | arXiv:2503.06416 中的获胜者保持私人草稿板；公开通道仅显示报价。 |
| 角色操纵 | "情绪姿态" | arXiv:2402.05863：通过绝望/紧急角色获得约 20% 的收益提升。 |
