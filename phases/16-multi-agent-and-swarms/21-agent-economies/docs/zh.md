# 智能体经济、代币激励与声誉

> 长时域自主智能体（METR 的 1 小时到 8 小时工作曲线）需要经济代理能力。新兴的**5 层堆栈**是：**DePIN**（物理计算）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产级智能体激励网络包括 **Bittensor**（TAO 子网奖励特定任务的模型）、**Fetch.ai / ASI 联盟**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于 Transformer 的工作量证明，将计算重新分配到生产性 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值信用归因**公平地奖励贡献智能体；Google Research 的"大语言模型的机制设计"提出了在单调聚合下的**代币拍卖**与第二价格支付。本课构建一个最小智能体市场，将 Shapley 值信用归因应用于多智能体流水线，并运行第二价格代币拍卖，使博弈论机制具体落地。

**语言：** Python（标准库）
**前置要求：** 第 16 阶段 · 16（谈判与讨价还价），第 16 阶段 · 09（并行/群体/网络架构）

## 问题

当智能体共同产生价值但需要单独获得奖励时，多智能体系统变得复杂。经典机制——平均分配、最后贡献者通吃——是不公平或可被博弈的。通过 Shapley 值的联盟式奖励在构造上是公平的，但计算成本高昂。2025-2026 年的文献推动了有用的近似：Shapley 采样、单调聚合拍卖以及从已确认贡献中累积的链上声誉。

除了信用归因，该领域已转向实际的经济智能体：Bittensor TAO 奖励挖矿计算以微调子网特定模型，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 使用，Gonka 将 Transformer 工作量证明重新分配到生产性 AI 任务。自主交易的智能体今天已经存在；问题在于如何对齐激励。

本课将智能体经济视为一个特定的问题家族——信用归因、机制设计和声誉——并用最少的数学构建每个部分，使思想深入人心。

## 概念

### 5 层智能体经济堆栈

1. **DePIN（物理计算）。** 出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。不是智能体特有的；智能体使用它。
2. **身份。** W3C 去中心化标识符（Decentralized Identifiers，DID）为每个智能体提供独立于任何平台的持久 ID。声誉累积到 DID 上。智能体网络协议（Agent Network Protocol，ANP）使用 DID 作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）允许智能体从自己的余额中支付 gas 费，而无需持有 ETH。智能体可以为服务、彼此或计算付费。
5. **治理。** 智能体 DAO：人类*和*智能体对协议变更进行投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都使用全部五层。Bittensor 使用 1、2、部分 3、部分 4，不使用 5。OpenAI 智能体只使用 3。该堆栈是参考地图，而非要求。

### Bittensor、Fetch.ai、Gonka——实际运行的内容

**Bittensor（TAO）。** 子网是专门的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其进行排名；权益加权评分分配 TAO 奖励。每个子网有自己的评估。经济教训：为特定任务的输出质量付费，而非使用的计算量。

**Fetch.ai / ASI 联盟。** ASI-1 Mini LLM 在 Fetch.ai 的网络上运行；用户用 FET 代币支付推理费用。这里的智能体即对等方的叙事更强：Fetch 上的一个智能体可以调用另一个智能体执行任务并用 FET 支付。

**Gonka。** Transformer 工作量证明："工作"是 Transformer 的前向传播。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来赚取收益。资源生产性的 PoW，而非基于哈希的 PoW。

截至 2026 年 4 月，这三者都是生产级的。收益分配各不相同。Bittensor 奖励相对于子网验证者的质量；Fetch 奖励由付费用户衡量的效用；Gonka 奖励可验证的推理工作。

### Shapley 值信用归因

三个智能体协作完成一个任务。输出得分为 0.8。谁贡献了什么？

Shapley 值：满足四个公理（效率性、对称性、线性性、空性）的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * 对所有排序 O 求和 (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是在排序 `O` 中 `i` 之前的智能体集合。实践中：枚举所有排列，记录每个智能体在每个排列中的边际贡献，取平均。

对于 N=3 个智能体，有 6 种排列。对于 N=10，有 360 万种——因此实践中你采样排序而非枚举。

### 用于聚合的第二价格拍卖

Google Research（"大语言模型的机制设计"）提出了用于聚合 LLM 输出的第二价格代币拍卖。设置：N 个智能体各自提出一个补全；每个智能体对被选中有一个私有价值。拍卖者选择最高价值的提案并支付*第二高*的价值。在单调聚合下（价值取决于哪个提案被选中，而非有多少个被出价），这是诚实的——智能体出价其真实价值。

为什么这对 LLM 系统重要：你可以将补全任务外包给具有不同定价的多个智能体；拍卖选择最佳并公平支付，智能体没有动机虚报。

### 声誉资本

一个绑定到 DID 的声誉分数从已确认的贡献中累积。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 读取成本低，用于路由决策（"将困难任务发送给高声誉智能体"）。
- 伪造成本高（随时间累积，绑定到 DID）。
- 可以被削减：未通过验证的贡献会扣分。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 值信用归因和一个简单的拍卖机制。关键主张：去中心化信用归因步骤使系统可审计且免疫于单点操纵。

### 经济学失效的地方

- **价格预言机操纵。** 如果信用函数可以被博弈，智能体就会博弈它。每个机制都需要对抗性测试。
- **女巫攻击。** 一个运营者启动 N 个假智能体来夸大自己的贡献。DID 减缓但不能阻止这一点；声誉的伪造成本是缓解措施。
- **验证成本。** 信用归因只有在验证者公平的情况下才公平。如果验证便宜（小型 LLM），它可以被博弈；如果昂贵（人类小组），系统无法扩展。
- **监管悬而未决。** 智能体经济与金融监管交叉。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法管辖区都处于法律灰色地带。

### 智能体经济何时有意义

- **具有异质运营者的开放网络。** 没有单一团队控制所有智能体。
- **可验证的输出。** 没有验证，信用归因就是猜测。
- **长时域工作流。** 一次性任务不会从声誉累积中受益。
- **代币化支付在你的司法管辖区法律上可行。**

在封闭的企业系统中，经济学让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 交付它

在 2026 年运行智能体经济：

- **从声誉开始，而非代币。** 声誉实现成本低且单独就有价值；代币增加了法律和经济复杂性。
- **先验证再奖励。** 永远不要在没有独立验证步骤的情况下分配信用。自我报告的质量会滋生女巫博弈。
- **Shapley 采样，而非 Shapley 精确。** 采样 100-1000 个排序；精确枚举无法扩展。
- **限制衰减因子和声誉下限。** 无界衰减会抹去合法贡献者；过慢的衰减奖励过时的高声誉智能体。
- **对抗性地审计机制。** 在开放网络之前运行红队场景。每个机制都有博弈论；你要找到漏洞，而非让攻击者找到。

## 练习

1. 运行 `code/main.py`。确认 Shapley 值之和等于总价值（效率性公理）。改变值函数；Shapley 分配是否按预期方向变化？
2. 实现 Shapley *采样*（对 K 个排序的蒙特卡洛）。K 如何影响近似精度？与 N=4 的精确值比较。
3. 在拍卖前实现一个联盟形成步骤：智能体可以合并成团队并作为一个单元出价。哪些联盟会形成？结果是否比单独出价帕累托更优？
4. 阅读 Google Research 的机制设计文章。找出一个假设，如果违反，会破坏诚实性。在 LLM 环境中，这种失败模式是什么样子的？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在合成任务上对 10 个智能体实现他们的 Shapley 步骤。精确计算需要多长时间？100 次抽样的采样有多接近？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|----------|
| DePIN | "去中心化物理基础设施" | 代币激励的计算/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | W3C 便携式 ID 规范。智能体声誉绑定到 DID，而非平台。 |
| ERC-4337 | "账户抽象" | 可以赞助 gas 的合约账户，使智能体支付成为可能。 |
| Shapley 值 | "公平信用归因" | 满足效率性、对称性、线性性、空性的唯一分配。 |
| 第二价格拍卖 | "Vickrey 拍卖" | 诚实机制：获胜者支付第二高出价。与单调聚合兼容。 |
| 声誉资本 | "累积的质量分数" | 绑定到 DID 的分数，来自已确认的贡献；随时间衰减。 |
| 智能体 DAO | "智能体 + 人类共同治理" | 智能体作为一等投票者的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU 积分 | "代币面额" | Bittensor TAO、Fetch.ai FET、各种 DePIN 代币。 |
