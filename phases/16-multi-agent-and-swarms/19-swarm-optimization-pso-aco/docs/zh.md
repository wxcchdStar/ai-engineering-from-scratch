# 面向 LLM 的群体优化（Swarm Optimization）——PSO、ACO

> 仿生优化正在 LLM 领域卷土重来。**LMPSO**（arXiv:2504.09247）使用 PSO，其中每个粒子的速度是一个提示词，LLM 生成下一个候选；在结构化序列输出（数学表达式、程序）上效果良好。**Model Swarms**（arXiv:2410.11163）将每个 LLM 专家视为模型权重流形上的 PSO 粒子，报告在 9 个数据集上仅用 200 个实例就比 12 个基线平均提升 **13.3%**。**SwarmPrompt**（ICAART 2025）混合 PSO + 灰狼优化用于提示词优化。**AMRO-S**（arXiv:2603.12933）是受 ACO 启发的信息素专家，用于多智能体 LLM 路由——**4.7 倍加速**，可解释的路由证据，质量门控异步更新将推理与学习解耦。本课程在提示词参数空间上实现 PSO，在智能体路由上实现 ACO，测量这些经典算法为何适合 LLM 时代，以及何时不适合。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 16 阶段 · 09（并行群体网络）、第 16 阶段 · 14（共识与 BFT）
**时间：** ~75 分钟

## 问题

你有一个在你的任务评估上得分为 62% 的提示词。你想改进它。天真的做法是无梯度手动调整，扩展性差。强化学习需要奖励信号和足够的 rollout 来训练。通过提示词进行反向传播实际上是不可能的——提示词是离散字符串，不是可微分参数。

经典仿生优化——用于连续搜索空间的 PSO，用于路径选择的 ACO——正是为这种机制设计的：无梯度、基于种群、每次评估便宜。将它们与 LLM 配对用于无梯度搜索步骤，你就得到了一个令人惊讶的实用优化器。

相同的模式适用于多智能体系统中的智能体*路由*。ACO 风格的信息素轨迹记录哪个智能体在哪种任务类型上效果最好，让路由器利用该轨迹，并衰减信息素以便可以重新发现路由。

## 概念

### PSO 复习（Kennedy & Eberhart 1995）

粒子群优化（Particle Swarm Optimization）：连续搜索空间中的粒子种群。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
评估 fitness(x_i)
如果改进则更新 p_best_i
如果是全局最佳则更新 g_best
```

其中 `p_best` 是粒子自身最佳，`g_best` 是群体最佳，`w, c1, c2` 是惯性 + 认知 + 社会权重，`r1, r2` 是随机因子。

### LLM 输出上的 PSO——LMPSO

arXiv:2504.09247 将 PSO 适配用于 LLM 生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个*提示词*，描述如何将当前输出修改为朝向个人/全局最佳。LLM 从速度提示词生成新输出。速度的"惯性"是一个提示词，如"进行小的增量更改"。

这在以下情况下效果良好：
-   输出是结构化的（可解析、可评估）。
-   适应度是自动的（测试运行、算术评估）。
-   种群较小（约 10-30 个粒子），因此总 LLM 调用保持可管理。

当适应度需要人工审查时效果不佳——每次迭代的成本变得过高。

### Model Swarms

arXiv:2410.11163 将 PSO 从输出层带到*模型*层。每个"粒子"是一个专家 LLM（参数）。群体通过无梯度更新将参数移向集体最佳。报告：在 9 个数据集上仅用每次迭代 200 个实例，比 12 个基线平均提升 13.3%。

关键洞察是 LLM 专家模型已经在共享参数流形（适配器权重、LoRA 增量）中接近。在这个低维子空间上的 PSO 便宜且有效。

### ACO 复习（Dorigo 1992）

蚁群优化（Ant Colony Optimization）：蚂蚁遍历图；每条路径有信息素轨迹。蚂蚁移动概率按信息素强度加权。完成任务的蚂蚁按解决方案质量比例沉积信息素。信息素随时间衰减。

### AMRO-S——用于智能体路由的 ACO

arXiv:2603.12933 使用 ACO 进行多智能体路由。每种任务类型是一个"目的地"；每个智能体是一条可能的路线。信息素加强产生良好输出的路线。关键贡献：

-   **可解释的路由证据。** 信息素强度是人类可读的信号。
-   **质量门控异步更新。** 信息素仅在质量检查通过后更新，将推理与学习解耦。
-   在多智能体路由基准测试上**4.7 倍加速**。

质量门很重要：没有它，快速但错误的智能体会积累信息素，系统锁定在错误的路由上。

### 何时为 LLM 使用 PSO / ACO

**使用 PSO 当：**
-   搜索空间是连续的或映射到连续参数（提示词嵌入、LoRA 权重、数值生成参数）。
-   适应度便宜且自动。
-   种群可以较小（10-30）。

**使用 ACO 当：**
-   你有路由或路径选择问题。
-   决策随时间加强（相同的任务类型会回来）。
-   你需要路由决策的可解释证据。

**不要使用两者当：**
-   适应度需要人工审查（每次迭代太昂贵）。
-   搜索空间是离散和组合的，PSO 无法覆盖（改用遗传算法）。
-   实时决策需要严格延迟（PSO/ACO 相对于单次启发式方法收敛缓慢）。

### 为什么仿生仍然胜出

基于梯度的方法需要可微分信号。LLM 输出和路由决策不是平凡可微的。伪梯度方法（强化学习路由器、DPO 风格提示词调优器）有效但需要昂贵的训练。

PSO 和 ACO 只需要一个*评估器*函数。如果你可以评分候选输出或路由决策，你就可以在空间上优化。这使得适用门槛低得多。

### 实际限制

-   **种群预算。** N 个粒子 × T 次迭代 × 每次评估成本。对于每次调用约 $0.02 的 LLM 评估，20 粒子 PSO 运行 50 次迭代成本约 $20。相应规划。
-   **探索 vs 利用。** 信息素衰减率和 PSO 惯性权衡；衰减太快 → 忘记解决方案；太慢 → 卡在早期局部最优。
-   **灾难性漂移。** 两种算法都可能收敛然后发散，如果适应度景观变化（新数据分布）。监控最佳适应度稳定性。

## 构建它

`code/main.py` 实现：

-   `LMPSO`——数值提示词参数（temperature、top_k 权重）上的 PSO。每个粒子的"LLM 生成"被模拟为脚本化适应度函数。运行算法 30 次迭代并显示 g_best 收敛。
-   `AMRO_S`——ACO 风格路由。3 个智能体，4 种任务类型，信息素矩阵，100 个路由任务。打印（任务类型 → 智能体选择）随时间分布以显示轨迹形成。
-   比较：相同任务流上的随机路由 vs ACO 路由。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
-   LMPSO：g_best 适应度在 30 次迭代中从随机改进到接近最优。
-   AMRO-S：信息素表在每种任务类型的正确智能体上稳定；ACO 路由在质量上比随机高约 30-40%，并且也减少延迟（更少重试）。

## 使用它

`outputs/skill-swarm-optimizer.md` 帮助在 PSO、ACO、遗传算法和基于梯度的优化器之间为 LLM / 智能体优化问题进行选择。

## 交付它

-   **从小开始。** 10-20 个粒子，20-50 次迭代。仅在收敛曲线显示明显增益时才扩大规模。
-   **记录每次迭代的信息素或 g_best。** 没有轨迹的群体优化器调试是痛苦的。
-   **质量门控更新。** 特别是对于 ACO 路由：快速但错误的智能体不得积累信息素。
-   **在分布变化时重置衰减。** 当你的评估分布变化时，老化的信息素是陈旧的；暂时重置或加倍衰减率。
-   **限制每次迭代成本。** 发出每次迭代成本指标。每次迭代花费 $500 且仅获得 0.5% 增益的 PSO 是不可交付的。

## 练习

1.  运行 `code/main.py`。观察 LMPSO 收敛。改变种群大小 5、10、20、50。在什么大小下收敛时间饱和？
2.  实现"灾难性漂移"实验：在第 30 次迭代后，改变适应度函数。PSO 适应多快？重置 `p_best` 有帮助吗？
3.  向 AMRO-S 添加质量门：仅在评估分数 > 0.7 的运行上沉积信息素。与无门控版本相比，这如何改变收敛？
4.  阅读 LMPSO（arXiv:2504.09247）。将论文的"速度作为提示词"映射回你的数值速度。模拟中丢失了什么，保留了什么？
5.  阅读 AMRO-S（arXiv:2603.12933）。实现带有异步信息素更新的解耦"推理快速路径"。在持续负载下，这如何改变系统延迟？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|----------|
| PSO | "粒子群优化" | Kennedy-Eberhart 1995。基于种群的无梯度优化器。 |
| ACO | "蚁群优化" | Dorigo 1992。通过信息素轨迹进行路径/路由优化。 |
| LMPSO | "带 LLM 生成的 PSO" | arXiv:2504.09247。速度是提示词；LLM 产生候选。 |
| Model Swarms | "专家权重上的 PSO" | arXiv:2410.11163。模型参数子空间上的无梯度更新。 |
| AMRO-S | "用于智能体路由的 ACO" | arXiv:2603.12933。任务类型 × 智能体的信息素矩阵。 |
| p_best / g_best | "个人 / 全局最佳" | 每个粒子和群体范围内迄今为止找到的最佳解决方案。 |
| 信息素 | "路由记忆" | 边上的强度；随时间衰减；按质量沉积。 |
| 质量门控更新 | "仅从好的运行中学习" | 以质量检查为条件的信息素沉积。 |
| 灾难性漂移 | "分布变化" | 适应度景观变化；旧的 p_best 和信息素变得陈旧。 |
