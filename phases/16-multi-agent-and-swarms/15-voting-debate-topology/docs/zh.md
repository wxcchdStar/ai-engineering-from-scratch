# 投票、自一致性（Self-Consistency）与辩论拓扑（Debate Topology）

> 最便宜的聚合：对 N 个独立智能体采样，多数投票。Wang 等人 2022 年的自一致性（Self-Consistency）通过对一个模型采样 N 次来实现。多智能体通过**异构**智能体扩展它，以摆脱单一文化（monoculture）——不同的模型、不同的提示词、不同的温度、不同的上下文。超越多数投票，辩论拓扑很重要：MultiAgentBench（arXiv:2503.01935, ACL 2025）评估了星形（star）/链形（chain）/树形（tree）/图（graph）协调，发现**图最适合研究**，超过约 4 个智能体后出现"协调税（coordination tax）"。AgentVerse（ICLR 2024）记录了两种涌现模式——志愿者行为（volunteer behaviors）和从众行为（conformity behaviors）——从众既是特性（找到共识）也是风险（群体思维，Groupthink，第 24 课）。本课程映射拓扑空间，构建每种变体，并测量协调税。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 16 阶段 · 07（心智社会与辩论）、第 16 阶段 · 14（共识与 BFT）
**时间：** ~75 分钟

## 问题

辩论可以提高准确性（Du 等人，arXiv:2305.14325）。它也可以降低准确性。辩论是否有帮助取决于四个结构性选择：

1.  谁与谁交谈（拓扑）。
2.  多少轮（Du 2023：轮数和智能体数量都独立重要）。
3.  智能体是否异构（不同的基础模型打破单一文化）。
4.  是否存在对抗性声音（钢人论证 vs. 稻草人论证）。

将"运行 5 个智能体并投票"附加到任务上的团队通常相对于单个智能体会退步。失败不是随机的。它们跟踪拓扑和异构性。本课程是拓扑地图。

## 概念

### 自一致性，单模型基线

Wang 等人 2022（"Self-Consistency Improves Chain of Thought Reasoning"）在 temperature > 0 下对同一模型采样 N 次，并对推理路径答案进行多数投票。GSM8K 上的结果：N=40 个样本比单次贪婪解码有显著增益。自一致性是多智能体投票的单智能体前身。

限制：自一致性使用一个基础模型。误差在构造上是相关的。如果模型有系统性偏差，所有 N 个样本共享它。

### 多智能体投票，异构扩展

用 N 个*不同*的智能体替换 N 个样本。不同的基础模型（Claude、GPT、Llama）、不同的提示词、不同的工具访问。好处：不相关误差。成本：不同的智能体成本不同；协调它们增加开销。

2026 年异构辩论的规范名称是 **A-HMAD**——对抗性异构多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。并非普遍采用，但论文使用该术语表示"不同模型辩论，减少来自单一文化崩溃的相关误差"。

### 四种拓扑

```
星形                链形               树形                图

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           （全连接）
```

星形：一个中心，所有其他只与中心交谈。等价于没有反向通道的监督者-工作者。
链形：线性，每个智能体看到前一个的输出。类似流水线。
树形：层级，由层级智能体系统使用（第 06 课）。
图：任意到任意。包括全连接团和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE, ACL 2025, arXiv:2503.01935）在包括研究、编码和规划的任务套件上对星形、链形、树形、图进行了基准测试。关键测量结果：

-   **图**拓扑在研究任务上胜出。信息任意到任意流动；智能体可以相互批判。
-   **星形**在快速答案事实性任务上胜出。中心过滤和整合。
-   **链形**在逐步流水线（分阶段细化）上胜出。
-   **协调税**在图拓扑中超过约 4 个智能体后出现。实际耗时和 token 成本增长快于质量。

4 智能体上限是经验的，不是根本的。它反映了 2026 年 LLM 上下文容量：每个智能体的上下文被同伴的输出填满，一旦每个人都能看到每个人，添加第 N+1 个智能体的边际价值下降。

### 多智能体辩论策略（"Should we be going MAD?"）

arXiv:2311.17371 是 2023 年 MAD 策略的调查。被其他人复现的关键发现：在结构上与自一致性相似（独立采样 + 聚合）的 MAD 变体在使用相同预算时通常不如自一致性。当智能体真正异构且辩论具有对抗性结构（一个智能体反对）时，MAD 最有帮助。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024）记录了两种从多智能体辩论中涌现的行为，即使没有显式设计：

-   **志愿者（Volunteer）。** 智能体主动提供帮助（"我可以做下一步"）。有用：它将工作分配给最适合子任务的智能体。
-   **从众（Conformity）。** 智能体调整其立场以匹配批判者，即使批判者是错误的。这是谄媚的辩论等价物（第 14 课）。

从众是为什么辩论直到达成一致会奖励霸凌者。有界轮次加上独立法官可以缓解。

### 异构性：真正推动准确性的旋钮

2024-2026 年实践文献中的一个模式：将你的 N 个智能体中的一个替换为不同的基础模型，比将 N 增加 1 带来更大的准确性提升。直觉是单一文化——每个新的独立误差源比额外的相关样本更有价值。

在极限情况下，异构性胜过数量。在大多数具有清晰真实答案的任务上，三个不同的模型胜过五个相同模型的副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中引用）形式化了"陪审团"——一小组专业化智能体，通过在每个阶段投票来细化答案。与简单的多数投票不同，陪审团有角色：一个智能体交叉审查，一个提供上下文，一个评分合理性。陪审团方法是简单投票（便宜，容易单一文化）和完整 MAD（昂贵，容易从众）之间的中点。

### 投票加辩论何时占主导

-   问题有真实答案（事实、数学、代码行为）。投票收敛是有意义的。
-   智能体可以访问不同的来源或工具（异构性可用）。
-   轮次有界（通常 2-3 轮），并且有独立的法官或验证器。
-   预算允许 3-5 个智能体。在图拓扑上超过 5-7 个，协调税占主导。

### 投票加辩论何时有害

-   问题是观点性的。智能体收敛到看起来最自信的答案，而不是最正确的。
-   所有智能体共享一个基础模型。单一文化使共识毫无意义。
-   轮次无界。从众每次都赢。
-   任务简单。单个智能体在 N=5 的自一致性下更便宜且同样准确。

## 构建它

`code/main.py` 实现：

-   `run_star(agents, hub, question)`——中心轮询每个工作者，聚合。
-   `run_chain(agents, question)`——顺序细化。
-   `run_tree(root, children, question)`——带深度-2 聚合的层级结构。
-   `run_graph(agents, question, rounds)`——全对全辩论，有界轮次。
-   脚本化异构性旋钮：每个智能体有一个 `error_bias` 指示其系统性错误。
-   一个测量工具，在 N=3, 5, 7 下运行每种拓扑，并报告（准确性、总 token、模拟实际耗时）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确性、token、延迟）的表格。图在研究型任务上 N=3-5 时胜出；星形在快速事实性任务上胜出；图在 N=7 时显示协调税（延迟膨胀快于准确性）。

## 使用它

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星形/链形/树形/图）、N（智能体数量）、异构性配置文件（要使用的基础模型）和轮次上限。

## 交付它

对于任何集成：

-   从**自一致性 N=5** 开始，使用一个强大的基础模型。这是便宜的基线。
-   如果准确性重要，升级到**异构投票 N=3**。测量增量。
-   仅在任务有结构（研究、多步骤）且可行有界轮次时，才升级到**辩论拓扑**。
-   始终记录少数簇。当少数持续正确时，你有一个多样性信号。
-   在准确性旁边基准测试实际耗时和 token。"以 10 倍成本获得更好的准确性"是一个商业决策。

## 练习

1.  运行 `code/main.py`。绘制图拓扑的协调税曲线：准确性 vs N，token vs N。曲线在哪个 N 处拐点？
2.  实现 A-HMAD：三个具有故意不同偏差的智能体。全相同偏差基线与 A-HMAD 在第 14 课的单一文化攻击上相比如何？
3.  在图拓扑中添加一个"法官"角色，它不投票，只对最终共识评分。这会改变涌现的从众行为吗？
4.  阅读 AgentVerse 论文（ICLR 2024）。识别你的实现最强烈地表现出哪种涌现行为。你能通过提示词更改引出相反的行为吗？
5.  阅读 MultiAgentBench（arXiv:2503.01935）第 4 节（拓扑实验）。使用你的工具在论文中的一个任务上复现"图胜研究"的结果。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|----------|
| 自一致性 | "采样 N 次，投票" | Wang 2022。单模型，N 个 temperature>0 样本，对推理路径多数投票。 |
| 异构性 | "不同的模型" | 不同基础模型或提示词家族的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体在轮次中交换批判的通用术语。见 Du 2023。 |
| A-HMAD | "对抗性异构 MAD" | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| 拓扑 | "谁与谁交谈" | 星形、链形、树形、图。决定信息流。 |
| 协调税 | "收益递减" | 在图拓扑上超过约 4 个智能体，成本增长快于质量。 |
| 志愿者行为 | "主动帮助" | AgentVerse 涌现模式：智能体主动承担步骤。 |
| 从众行为 | "压力下的同意" | AgentVerse 涌现模式：智能体与批判者对齐。 |
| 陪审团 | "小型专业化小组" | Sibyl 风格集成，具有角色（审查员、上下文、评分员）。 |
