# 心智社会（Society of Mind）与多智能体辩论（Multi-Agent Debate）

> Minsky 1986 年的前提——智能是一个由专家组成的社会——每十年被重新发现一次。2023 年，Du 等人将其转化为一个具体算法：多个 LLM 实例提出答案，阅读彼此的答案，进行批判，并更新。经过 N 轮，它们收敛到一个共识，在六项推理和事实性任务上击败了零样本 CoT 和自我反思（Self-Reflection）。两个发现很重要：**多个智能体**和**多轮**都独立地做出贡献。社会胜过单智能体独白；多轮交换胜过一次性投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 16 阶段 · 04（原语模型）
**时间：** ~60 分钟

## 问题

自一致性（Self-consistency）——对一个模型多次采样并取多数答案——是你可以附加的最便宜的推理改进。它有效，但很快饱和。你可以将样本加倍，却看不到另一个有意义的跃升。

辩论打破了饱和。不是来自一个模型的 N 个独立样本，而是 N 个智能体阅读彼此的推理并修正。样本之间的相关性下降（它们不再是独立同分布），收敛点通常在独立同分布投票自信地出错时是正确的。

## 概念

### Du 等人 2023 算法

来自 arXiv:2305.14325（ICML 2024）：

1.  N 个智能体中的每一个对问题产生一个初始答案。
2.  对于第 r = 2..R 轮：每个智能体看到其他智能体第 r-1 轮的答案，并被要求"考虑到这些，给出你更新的答案。"
3.  R 轮后，对最终答案进行多数投票。

该论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行了测试。辩论始终优于 CoT 和自我反思。

### 两个独立的旋钮

同一篇论文的消融实验：

-   **仅智能体数量**（1 轮，N 的多数投票）在大多数任务上优于单智能体，但趋于平稳。
-   **仅轮数**（1 个智能体看到自己先前的推理）几乎没有帮助——反思的已知弱点。
-   **两者结合**产生大的跃升。智能体之间的多轮交换驱动了增益。

### 为什么它有效

两个机制：

1.  **暴露于分歧。** 当一个智能体看到另一个智能体具有不同结论的推理链时，它必须要么证明自己，要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2.  **相关误差减少。** 在自一致性中，所有样本来自同一模型，因此误差相关——你平均到一个自信地错误的答案。不同的模型或不同的种子去相关。不同的*辩论观点*进一步去相关。

### 异构辩论（Heterogeneous debate）

A-HMAD 及相关后续研究为不同的智能体使用*不同的基础模型*。Llama + Claude + GPT 辩论减少了单一文化崩溃（Monoculture collapse，第 26 课），因为一个模型家族的相关误差不被其他家族共享。

缺点：参与辩论的弱模型可能将共识拖向其错误答案（参见"Should we be going MAD?"，arXiv:2311.17371）。

### NLSOM——129 智能体扩展

Zhuge 等人（"Mindstorms in Natural Language-Based Societies of Mind," arXiv:2305.17066）将这一想法扩展到 129 个成员的社会。结果是：专业化和自组织随着规模而出现，系统在视觉问答等任务上优于单智能体。

### 故障模式

-   **谄媚级联（Sycophancy cascade）。** 所有智能体都服从听起来最自信的那个。辩论崩溃为最响亮的声音。提示对抗性角色（"一个智能体必须论证反方立场"）有帮助。
-   **主题漂移（Topic drift）。** 多轮辩论偏离原始问题。缓解措施：每轮重新注入问题。
-   **计算爆炸。** N 个智能体 × R 轮 = N·R 次 LLM 调用，每次调用上下文都在增长。5 智能体、5 轮辩论是 25 次调用，上下文不断增长。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## 构建它

`code/main.py` 在一个数学问题上运行 3 智能体 × 3 轮辩论，每个智能体以不同的（可能错误的）答案开始。智能体是脚本化的——每个通过按脚本化置信度加权平均邻居的答案来"更新"。收敛在逐轮日志中可见。

演示显示了两个关键效果：

-   单轮交换使智能体更接近正确答案。
-   超过第 2 轮的额外轮次显示收益递减（与 Du 等人的平台期一致）。

运行：

```
python3 code/main.py
```

## 使用它

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮数、异构性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗性）。它还在运行前估计 token 成本。

## 交付它

如果你交付辩论：

-   **轮数上限为 3。** Du 等人显示 3 轮捕获了大部分增益。更多是成本，不是质量。
-   **智能体数量上限为 5。** 超过 5，上下文膨胀和成本占主导。
-   **默认异构。** 池中至少有两个不同的基础模型。
-   **对抗性槽位。** 一个智能体被提示无论如何都要不同意。打破谄媚。
-   **记录每一轮。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习

1.  运行 `code/main.py`，然后将轮数设置为 5 并观察收益递减。额外收敛在哪一轮停止？
2.  添加第四个具有对抗性角色的智能体：总是不同意当前多数。这会破坏还是改善收敛？
3.  绘制（打印）每轮的一致性分数（多数答案上的智能体比例）。它何时达到 1.0，这等同于"正确"吗？
4.  阅读 Du 等人第 4 节消融实验。使用此代码复现"仅智能体"vs"仅轮数"vs"两者"的结果。
5.  阅读"Should we be going MAD?"（arXiv:2311.17371）并列出两种超越轮转的辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|----------|
| 心智社会（Society of Mind） | "Minsky 的想法" | 作为交互专家的智能；1986 年的框架现在通过 LLM 辩论得以操作化。 |
| 多智能体辩论 | "智能体争论" | N 个智能体提出、相互批判、在 R 轮中修正，多数投票。 |
| 共识（Consensus） | "它们同意了" | 不是认识论真理——只是多数答案上的比例。可能是自信地错误的。 |
| 轮数（Rounds） | "交换步骤" | 一轮 = 每个智能体读取其他智能体并更新一次。 |
| 异构辩论 | "混合模型家族" | 使用不同的基础模型来去相关误差。 |
| 谄媚级联 | "每个人都同意声音最大的那个" | 辩论失败，智能体服从最自信的智能体，无论正确性如何。 |
| NLSOM | "129 智能体社会" | 自然语言心智社会；Zhuge 等人的扩展版本。 |
| 相关误差 | "相同模型，相同 bug" | 为什么自一致性饱和；跨不同观点的辩论去相关。 |
