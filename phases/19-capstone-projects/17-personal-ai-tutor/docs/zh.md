# 实战项目 17 — 个人 AI 导师（自适应、多模态、带记忆）

> Khanmigo（Khan Academy）、Duolingo Max、Google LearnLM / Gemini for Education、Quizlet Q-Chat 和 Synthesis Tutor 在 2026 年都规模化地提供了自适应多模态辅导。其共同形态是：苏格拉底式策略（绝不直接给出答案）、每次交互后更新的学习者模型（贝叶斯知识追踪风格）、语音 + 文本 + 拍照数学输入、课程图谱检索、间隔重复调度，以及针对适龄内容的严格安全过滤器。本实战项目要求交付一个特定学科的导师（K-12 代数或 Python 入门），对 10 名学习者进行为期两周的效能研究，并通过内容安全审计。

**类型：** 实战项目（Capstone）
**语言：** Python（后端、学习者模型）、TypeScript（Web 应用）、SQL（课程图谱通过 Postgres + Neo4j）
**前置课程：** 第 5 阶段（NLP）、第 6 阶段（语音）、第 11 阶段（LLM 工程）、第 12 阶段（多模态）、第 14 阶段（智能体）、第 17 阶段（基础设施）、第 18 阶段（安全）
**涉及阶段：** P5 · P6 · P11 · P12 · P14 · P17 · P18
**时间：** 30 小时

## 问题

自适应辅导曾经是教育科技的研究利基。到 2026 年，它已成为消费产品。Khanmigo 已部署在美国大多数学区。Duolingo Max 达到了数千万月活用户。Google 的 LearnLM / Gemini for Education 为 Google Classroom 中的辅导提供支持。Quizlet Q-Chat 与闪卡并排放置。Synthesis Tutor 以面向好奇儿童的导师实现了病毒式传播。共同要素：多模态输入（打字、说话、拍照方程式）、苏格拉底式教学法（先问后解释）、每次交互后更新的学习者模型，以及严格的适龄安全。

你将为一个特定群体构建一个这样的系统。衡量标准是实际的效能研究：10 名学习者为期两周的前测和后测分数。语音循环必须感觉自然（实战项目 03 的子栈）。记忆必须尊重隐私。安全过滤器必须通过面向 K-12 的 COPPA 感知红队测试。

## 概念

四个组件。**导师策略**是一个苏格拉底式循环：当学习者要求答案时，策略提出引导性问题；当他们答对时，进入下一个概念；当他们卡住时，提供脚手架式提示。**学习者模型**是贝叶斯知识追踪（或简单变体），在每次交互后更新每个课程节点的掌握概率。**课程图谱**是一个 Neo4j 的概念图，带有前置边；策略遍历图谱以选择下一个概念。**记忆**是一个情节 + 语义存储（agentmemory 风格），保存过去的交互、错误和偏好。

UX 是多模态的。文本输入用于打字答案。语音输入通过 LiveKit + Whisper（重用实战项目 03）。拍照输入用于数学问题，通过 dots.ocr 或 PaliGemma 2。语音输出通过 Cartesia Sonic-2。安全使用 Llama Guard 4 加上适龄过滤器（阻止成人内容、暴力、自残）和 COPPA 感知的记忆保留策略。

效能研究是交付物。10 名学习者，前测和后测，两周。报告学习增益差异和置信区间。与非自适应基线（相同内容线性交付，无导师策略）进行比较。

## 技术栈

- 学科选择：K-12 代数或 Python 入门（选择一个深入）
- 导师策略：LangGraph 配合 Claude Sonnet 4.7（带提示缓存）
- 学习者模型：贝叶斯知识追踪（经典）或 FSRS 用于间隔
- 课程图谱：Neo4j 的概念 + 前置边 + OER 内容
- 记忆：agentmemory 风格持久向量 + 情节 + 语义存储
- 语音：LiveKit Agents 1.0 + Cartesia Sonic-2（重用实战项目 03 子栈）
- 拍照数学：dots.ocr 或 PaliGemma 2 用于方程式识别
- 安全：Llama Guard 4 + 自定义适龄过滤器
- 评估：Bloom 级别问题生成、前/后测套件、效能研究工具

## 构建步骤

1. **课程图谱。** 构建一个包含 50-150 个概念节点的 Neo4j 图谱（例如，K-12 代数从"数轴"到"二次公式"），带有前置边。为每个节点附加 OER 内容（Open Textbook、OpenStax）。

2. **学习者模型。** 使用先验初始化贝叶斯知识追踪：猜测、滑移、学习率。每次交互后更新每概念掌握度。按学习者持久化。

3. **导师策略。** LangGraph 节点：`read_signal`（学习者的答案是正确的/部分的/卡住的？）、`select_concept`（遍历课程图谱选择最高优先级概念）、`scaffold`（苏格拉底式提示）、`update_mastery`。

4. **记忆。** 每次交互写入情节存储。错误和偏好提升到语义记忆。COPPA 感知保留策略：1 年后自动删除，家长可访问。

5. **语音路径。** 附加到导师策略的 LiveKit Agents 工作器。ASR 通过 Whisper-v3-turbo。TTS 通过 Cartesia Sonic-2。支持打断（重用实战项目 03 机制）。

6. **拍照数学路径。** 上传或拍摄图像；运行 dots.ocr 或 PaliGemma 2 识别方程式；作为结构化输入馈送给导师。

7. **安全。** 每个模型输出通过 Llama Guard 4 + 适龄过滤器（阻止自残、成人内容、暴力）。记忆访问由学习者 ID 范围限定；家长访问表面用于删除。

8. **效能研究。** 10 名学习者，前测（标准化 30 题基线），两周导师交互（每周 3 次），后测。与相同内容上的 10 名学习者非自适应基线队列进行比较。

9. **每周进度报告。** 每学习者，自动生成 PDF 摘要，包含探索的主题、掌握轨迹和推荐的下一步。

## 使用方式

```
学习者："我不明白为什么 3x + 6 = 12 意味着 x = 2"
[信号]   卡住
[概念]   '分离变量'（前置：加法-减法-等式）
[脚手架] "你会在两边减去什么数字来开始？"
学习者："6"
[信号]   正确
[掌握度] 加法-减法-等式：0.62 -> 0.77
[概念]   继续'分离变量'
[脚手架] "很好。现在 3x / 3 等于什么？"
```

## 练习

1. 在有和没有自适应学习者模型（随机概念顺序）的情况下运行效能研究。报告差异。预期自适应会胜出，但幅度是有趣的数字。
2. 添加多模态探测：相同概念问题以文本、语音和拍照方式交付。测量学习者是否以他们偏好的模态更快收敛。
3. 构建家长仪表盘：练习的主题、掌握轨迹、即将到来的概念、安全事件（任何防护栏触发）。COPPA 对齐。
4. 添加语言切换模式：导师接受西班牙语输入并用西班牙语教学。测量 X-Guard 覆盖。
5. 压力测试记忆隐私：验证学习者 A 无法看到学习者 B 的数据，即使通过语音片段重新摄入攻击。记录尝试的访问并告警。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------------|------------------------|
| 苏格拉底式策略 | "提问，而非直接给出" | 导师提出引导性问题而非给出答案 |
| 贝叶斯知识追踪 | "BKT" | 用于每概念掌握概率的经典学习者模型方程 |
| FSRS | "自由间隔重复调度器" | 2024 年间隔重复调度器，优于 SM-2 |
| 课程图谱 | "概念 DAG" | Neo4j 的概念图，带有前置边 |
| 情节记忆 | "每交互日志" | 每次交互存储以供后续检索 |
| 语义记忆 | "学到的模式存储" | 从情节记忆提升的压缩错误和偏好 |
| COPPA | "儿童隐私法" | 美国法律，限制从 13 岁以下儿童收集数据 |
