# 实战项目 05 — 自主研究智能体（AI-Scientist 级别）

> Sakana 的 AI-Scientist-v2 发表了完整论文。Agent Laboratory 运行了实验。Allen AI 分享了追踪记录。2026 年的形态是：对实验进行规划-执行-验证树搜索，预算成本控制，沙箱化代码执行，视觉反馈 LaTeX 写作器，以及自动化的 NeurIPS 风格审稿人组合。本实战项目是构建一个这样的智能体，在每篇论文 30 美元的预算内端到端运行，并通过 Sakana 记录的沙箱逃逸红队测试。

**类型：** 实战项目（Capstone）
**语言：** Python（智能体 + 沙箱）、LaTeX（输出）
**前置课程：** 第 2 阶段（机器学习）、第 3 阶段（深度学习）、第 7 阶段（Transformers）、第 10 阶段（从零构建 LLM）、第 14 阶段（智能体）、第 15 阶段（自主系统）、第 16 阶段（多智能体）、第 18 阶段（安全）
**涉及阶段：** P0 · P2 · P3 · P7 · P10 · P14 · P15 · P16 · P18
**时间：** 40 小时

## 问题

自主研究智能体在 2026 年跨越了一个门槛。Sakana AI 的 AI-Scientist-v2 在 Nature 上发表，其生成的论文通过了研讨会同行评审。ShinkaEvolve（ICLR 2026）将这一路线扩展到演化假设。AMD 的 Agent Laboratory 发布了可复现的追踪记录。这些智能体并非魔法——它们是一个规划-执行-验证循环，在候选实验树上运行，配有成本上限、种子绑定的沙箱和自动化评审。其精髓在于循环、预算和安全叙述。

你将通过在狭窄领域（例如，在 1 亿参数 transformer 上进行注意力稀疏性消融实验）中针对一个种子想法实现该循环来学习。价值不在于第一次运行就发现新东西。价值在于基础设施：树搜索、实验沙箱、写作器-审稿人循环、红队报告。Sakana 团队记录了沙箱逃逸失败案例；你的智能体必须通过相同的红队测试。

## 概念

该智能体是一个最佳优先树搜索。节点是实验规格：（假设、配置、代码、预期结果）。扩展步骤提出带有小修改的子节点（更换优化器、调整批量大小、消融某个组件）。每个子节点在一个全新的沙箱中运行，配有硬性资源上限。结果反馈给评分函数，该函数按（新颖性 × 质量 × 剩余预算）对节点进行排名。树不断增长直到预算耗尽，然后最佳分支被撰写成文。

写作器是多模态的。它生成 LaTeX 草稿，编译，渲染图表，并将渲染后的 PDF 反馈给 Claude Opus 4.7 的视觉模式，以对布局、图表可读性和声明-证据对齐进行 critique（批评性审查）。一个由五个 LLM 评委组成的审稿人组合给出 NeurIPS 风格的分数（新颖性、严谨性、清晰性、可复现性、影响力）；如果平均分低于阈值，论文将带着批评意见返回给写作器。

安全是承重的。每个实验在 E2B 或 Daytona 沙箱中运行，无网络出口、有界墙钟时间和固定资源限制。智能体的代码生成步骤通过一个策略层，该层阻止逃逸沙箱的系统调用。红队报告复现 Sakana 记录的攻击面（fork 炸弹、文件系统逃逸、LLM 编写的网络调用）。

## 技术栈

- 编排：LangGraph 配合检查点和人工审批门控
- 树搜索：自定义最佳优先搜索，在实验节点上运行（Sakana v2 的 AB-MCTS 风格）
- 沙箱：每个实验使用 E2B，Docker-in-Docker 作为备选；通过 cgroups 设置资源上限
- 文献：Semantic Scholar Graph API + OpenAlex + 本地 FAISS 摘要缓存
- 写作器：LaTeX 模板 + Claude Opus 4.7（视觉模式）用于图表 critique 和布局
- 审稿人：5 个评委的组合（Opus 4.7、GPT-5.4、Gemini 3 Pro、DeepSeek R1、Qwen3-Max），加权聚合
- 实验框架：PyTorch 2.5 用于物理实验，W&B 用于日志记录
- 可观测性：Langfuse 用于智能体追踪，每篇论文 30 美元硬性预算

## 构建步骤

1. **种子和领域范围界定。** 取一个种子想法（例如，"研究 10 亿参数以下 transformer 注意力图中的稀疏模式"）。定义搜索空间：模型、数据集、计算预算。

2. **文献检索。** 查询 Semantic Scholar + OpenAlex 获取 50 篇最相关的高引用论文；本地缓存摘要；生成 1 页领域摘要。

3. **树脚手架。** 使用种子假设初始化根节点。实现 `expand(node) -> children`，提出小修改建议（每个子节点一个配置变更）。实现 `score(node)` 作为加权的新颖性 × 质量 × 预算项。

4. **沙箱包装。** 每个实验运行 `docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only`（或等效的 E2B 策略）。种子写入沙箱；输出以只读方式挂载出来。

5. **规划-执行-验证循环。** `plan` 提出子节点。`execute` 运行沙箱，捕获日志和指标。`verify` 对指标运行单元检查（损失是否下降？消融是否隔离了效应？）。失败的节点在树上存储失败原因。

6. **写作器。** 预算用完后，选择最佳分支。使用 matplotlib 渲染图表。通过 Claude Opus 4.7 生成 LaTeX 草稿，上下文中包含分支追踪。编译。将编译后的 PDF 反馈给 Opus 4.7 视觉模式进行 critique。迭代。

7. **审稿人组合。** 五个评委按 NeurIPS 风格评分标准对草稿进行评分（新颖性、严谨性、清晰性、可复现性、影响力）。如果平均分 < 4.0/5，将草稿返回给写作器并附上批评意见。最多重写 3 次后硬性停止。

8. **红队。** 构建或集成一组针对沙箱的对抗性任务：fork 炸弹、网络外泄尝试、文件系统逃逸、LLM 编写的 shell 元字符。确认所有攻击均被阻止。撰写发现报告。

9. **可复现性。** 每篇论文附带其树搜索追踪 JSON、种子、W&B 运行链接、沙箱配置以及端到端复现的 README。

## 交付标准

`outputs/skill-ai-scientist.md` 是交付物。给定一个种子想法 + 一个领域 + 30 美元预算，它运行完整管道并输出一篇可评审的论文以及可复现性捆绑包。

| 权重 | 标准 | 衡量方式 |
|:-:|---|---|
| 25 | 论文质量 | 与已发表研讨会论文的盲审评分对比 |
| 20 | 实验严谨性 | 基线、种子、消融实验；每个声明都有结果表中的单元格支持 |
| 20 | 成本和计算纪律 | 每篇论文 30 美元上限强制执行，Langfuse 追踪 |
| 20 | 安全 | 沙箱红队通过；网络策略和终止开关已验证 |
| 15 | 可复现性 | 使用相同种子的一键重跑可复现论文 |
| **100** | | |

## 练习

1. 在同一领域针对三个不同的种子想法运行管道。比较树搜索的哪些部分重叠。识别重复浪费的计算。
2. 在实验执行前，为估计成本超过 5 美元的节点添加人机协同（Human-in-the-Loop）门控。测量总成本下降了多少。
3. 将审稿人组合替换为单个评委。在一组已知差论文的保留集上测量误接受率。
4. 引入网络外泄红队测试：智能体编写尝试 `curl` 外部地址的代码。确认 `--network=none` 策略阻止了该尝试。记录该尝试。
5. 将你的树搜索与平坦随机基线（相同预算，无扩展策略）进行对比。报告新颖性 × 质量的增益。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------------|------------------------|
| 树搜索 | "AB-MCTS 风格扩展" | 在实验节点上进行最佳优先探索，使用新颖性×质量×预算评分 |
| 沙箱 | "实验隔离" | 无网络、有界 CPU/内存、固定种子、只读输入的容器 |
| 视觉 critique | "渲染后阅读" | 将论文编译为 PDF，将 PDF 反馈给 VLM 进行布局和声明-证据 critique |
| 审稿人组合 | "自动化同行评审" | 多个 LLM 评委使用 NeurIPS 评分标准对论文评分；加权聚合门控管道 |
| 新颖性评分 | "这是新的吗？" | 惩罚与 50 篇论文文献缓存接近程度的启发式方法 |
| 成本上限 | "$ 预算" | 每篇论文总花费的硬性上限；Langfuse 计数器 + 运行前估算 |
| 红队 | "沙箱逃逸审计" | 如果策略有误，将逃逸沙箱的对抗性任务 |
