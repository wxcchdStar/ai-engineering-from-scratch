# LLM 的差分隐私

> DP-SGD 仍然是标准——注入噪声的梯度更新提供形式化的 (epsilon, delta) 保证。计算、内存和效用的开销很大；参数高效的 DP 微调（LoRA + DP-SGD）是 2025 年的常见配置（ACM 2025）。两组证据存在张力：基于金丝雀的成员推断（Duan 等人，2024）报告对语言模型的成功有限；训练数据提取（Carlini 等人，2021；Nasr 等人，2025）恢复了大量逐字记忆。解决方案（arXiv:2503.06808，2025 年 3 月）：差距在于测量内容——插入的金丝雀 vs"最可提取"的数据。新的金丝雀设计使得无需影子模型即可进行基于损失的 MIA，并产生了对在真实数据上训练的 LLM 进行具有现实 DP 保证的首次非平凡 DP 审计。替代方案：PMixED（arXiv:2403.15638）——在推理时通过专家混合在下一个 token 分布上进行私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过 LLM 反馈的差分隐私逆转（Differential Privacy Reversal via LLM Feedback）——置信度分数泄露。

## 学习目标

- 定义 (epsilon, delta)-差分隐私并陈述 DP-SGD 配方。
- 解释 2024-2025 年的张力：金丝雀 MIA vs 训练数据提取给出不同的图景。
- 描述 PMixED 以及为什么推理时私有预测是 DP 训练的替代方案。
- 描述通过 LLM 反馈的差分隐私逆转攻击。

## 问题

LLM 会记忆。Carlini 等人 2021 表明生产语言模型按需复现逐字训练文本。DP 是形式化防御：训练使得输出在可证明的程度上对任何单个训练示例不敏感。2024-2025 年的证据显示 DP-SGD 是必要的，但部署的 ε 值可能与威胁模型不匹配。

## 概念

### (ε, δ)-差分隐私

一个随机算法 M 是 (ε, δ)-DP 的，如果对于任何相差一个示例的两个数据集和任何事件 S：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ。

解释：输出分布足够接近（由 ε 参数化），以至于任何单一个体的贡献不能被可靠推断，除非以概率 δ。

### DP-SGD

Abadi 等人 2016。标准配方：
1. 采样一个小批量。
2. 计算每个示例的梯度。
3. 将每个示例的梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和并添加标准差为 σ * C 的高斯噪声。
5. 使用带噪声的和来更新参数。

隐私成本由会计器（Moments Accountant、Rényi DP accountant）跟踪。LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而异；没有普遍"安全"的默认 ε。已发表的示例在 LLM 训练设置中大致跨越 ε ≈ 1–10，但这些是说明性的——不是推荐的默认值。较低的 ε 通常需要更多噪声，可能增加效用损失。

### LoRA + DP-SGD

前沿模型的完整 DP-SGD 是禁止性的。LoRA（Hu 等人 2022）将梯度更新限制在一个小适配器上，减少了每个示例的梯度存储。LoRA + DP-SGD 是 2025 年的常见配置。DP 保证适用于适配器；基础模型保持固定。

### 2024-2025 年的张力

两条证据线：

- **金丝雀 MIA（Duan 等人 2024）。** 在训练数据中插入独特的金丝雀，测量成员推断攻击者是否能识别它们。报告对语言模型的成功有限。暗示 MIA 很难。
- **训练数据提取（Carlini 2021，Nasr 等人 2025）。** 用前缀提示模型；测量它是否从训练中恢复逐字文本。报告大量记忆。暗示 MIA 在相关意义上是容易的。

2025 年 3 月解决方案（arXiv:2503.06808）：两者测量不同的东西。MIA 问"示例 e 在 D 中吗？"针对插入的金丝雀。提取问"我能从 D 中恢复什么？""最可提取"的示例对隐私才是重要的；金丝雀低估了这一点，因为它们没有被优化为可提取的。

新的金丝雀设计。无需影子模型的基于损失的 MIA。对在真实数据上训练的 LLM 进行具有现实 DP 保证的首次非平凡 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时私有预测。在下一个 token 分布上进行专家混合；每个专家看到训练数据的一个分片；聚合添加噪声以实现 DP。完全避免 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两者都以不同的威胁模型为代价回避了完整 DP 训练的效用成本。

### 通过 LLM 反馈的差分隐私逆转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言机来重新识别个体。即使输出不泄露，置信度分布也可能泄露。

防御：不暴露置信度，或在暴露前截断/量化它们。这是 (ε, δ)-DP 训练之外的额外要求。

## 使用它

`code/main.py` 在玩具二分类数据集上模拟 DP-SGD。你可以扫描噪声乘数 σ 和裁剪范数 C，并跟踪 (ε, δ) 预算和准确率代价。"金丝雀攻击"插入一个独特的训练示例，并测量对数损失测试在 DP 前后是否能检测到它。

## 交付它

本课产出 `outputs/skill-dp-audit.md`。给定一个语言模型部署的 DP 声明，它审计：(ε, δ) 值、使用的会计器、MIA 评估协议，以及是否评估了置信度暴露向量。

## 练习

1. 运行 `code/main.py`。在 {0.5, 1.0, 2.0} 中扫描 σ，报告 (ε, δ)-准确率权衡。找出效用崩溃的点。

2. 实现金丝雀插入和对数损失测试。在 σ = 1.0 时测量 DP-SGD 前后的检测率。

3. 阅读 Nasr 等人 2025 关于训练数据提取的内容。为什么提取成功在中等 ε 下不崩溃？这对 MIA 作为评估意味着什么？

4. 设计一个完全在推理时使用 PMixED（arXiv:2403.15638）的部署。PMixED 解决而 DP-SGD 不解决的威胁模型是什么？

5. 草拟通过 LLM 反馈的 DP 逆转攻击。设计一个限制置信度分数泄露的对策，并估计其部署成本。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| DP | "(ε, δ)-差分隐私" | 形式化隐私：在相邻数据集变化下输出分布接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效私有微调" | 在低秩适配器上的 DP-SGD；标准 2025 配置 |
| MIA | "成员推断" | 确定一个示例是否在训练数据中的攻击 |
| 金丝雀（Canary） | "插入的水印示例" | 用于测量 DP 泄露的独特训练示例 |
| PMixED | "私有推理混合" | 通过专家混合在下一个 token 分布上进行推理时 DP |
| DP 逆转（DP Reversal） | "置信度泄露攻击" | 使用模型置信度作为预言机进行重新识别的攻击 |
