# 指令遵循作为对齐信号（Instruction Following as Alignment Signal）

> 后来对 RLHF 的每一项批评都在反驳这个流程。在你研究优化压力如何扭曲代理指标之前，你必须先看到这个代理指标。InstructGPT（Ouyang 等人，2022）定义了参考架构：在指令-响应对上进行监督微调，在成对偏好排序上训练奖励模型，以及使用 PPO 对奖励模型进行优化，并加上对 SFT 策略的 KL 惩罚。一个 1.3B 的 InstructGPT 比 175B 的 GPT-3 更受偏好。这一结果就是为什么 2026 年每个前沿实验室仍然在交付一个 RLHF 形态的后训练流程。

**类型：** 学习
**语言：** Python（标准库，玩具三阶段流程）
**前置要求：** 第 10 阶段 · 06（SFT），第 10 阶段 · 07（RLHF），第 10 阶段 · 08（DPO）
**时间：** 约 45 分钟

## 学习目标

- 说出 InstructGPT 流程的三个阶段以及每个阶段使用的损失函数。
- 解释为什么一个 1.3B 的指令微调模型在人类偏好评估中击败了原始 175B GPT-3。
- 说明第三阶段的 KL 惩罚在防止什么，以及为什么移除它会导致模式寻求（mode-seeking）行为。
- 描述对齐税（alignment tax）以及 Ouyang 等人用来对抗它的 PPO-ptx 缓解措施。

## 问题

预训练语言模型会补全文本。它们不会回答问题。问 GPT-3「写一个反转列表的 Python 函数」，你通常会得到另一个提示，因为大部分训练分布是会继续生成更多文本的网页文本。模型在完成它的工作——只是这个工作是错的。

每个严肃的实验室用来修复这个问题的代理指标是人类偏好。两个补全结果交给评分员；评分员选择更好的那个；奖励模型学习评分员的判断。然后一个 RL 循环将策略推向奖励模型评分高的输出。这就是完整的 InstructGPT 论文，用三句话概括。论文的其余部分是工程。

## 概念

### 第一阶段：监督微调（SFT）

收集提示-响应对，其中响应是一个善意的人类会写的内容。Ouyang 等人使用了来自标注员和 OpenAI API 的 13k 个提示。用标准交叉熵损失在这些数据上微调基础模型。

SFT 给你什么：模型现在回答问题而不是继续它们。它不给你什么：关于当多个答案都合理时评分员偏好哪个答案的任何信号。

### 第二阶段：奖励模型（RM）

对于每个提示，从 SFT 模型中采样 K 个补全结果。标注员对它们进行排序。训练一个奖励模型，对任何提示-响应对进行评分，使得对于 `y_w` 被偏好于 `y_l` 的对：

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

这是 Bradley-Terry 成对偏好损失。RM 通常从 SFT 模型初始化，将 LM 头替换为标量头。

奖励模型很小：对于 175B 的 InstructGPT，6B 就足够了。它们也很脆弱——论文第 5 节主要讨论在小规模下出现的奖励欺骗（reward-hacking）行为。

### 第三阶段：带 KL 惩罚的 PPO

定义目标：

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

用 PPO 最大化。KL 项防止 `pi` 偏离 SFT 策略太远。没有它，优化器会找到对抗样本——在 RM 下得分高的字符串，因为 RM 从未见过它们，而不是因为人类真的偏好它们。

KL 系数 `beta` 是最重要的 RLHF 超参数。太低：奖励欺骗。太高：相比 SFT 没有改进。

### 对齐税

RLHF 之后，模型更受人类偏好，但在标准基准（SQuAD、HellaSwag、DROP）上退化。Ouyang 等人称之为对齐税，并用 PPO-ptx 修复：将预训练梯度混入 RL 目标，使模型不会忘记它从未被奖励过的下游任务。

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx 成为标准。Anthropic、DeepMind 和 Meta 都使用某种变体。

### 结果

一个 1.3B 的 InstructGPT（SFT + RM + PPO-ptx）在大约 70% 的情况下被标注员偏好于 175B 的基础 GPT-3。在生产流量的隐藏测试提示上差距更大。从这个数字可以读出两件事：

1. 对齐是与能力不同的轴。175B 模型有更多能力；1.3B 模型有更多对齐；标注员偏好对齐的那个。
2. 能力下限由基础模型设定。你不能通过 RLHF 让一个基础模型知道它从未见过的事实。

### 为什么这是第 18 阶段的参考点

后续课程中的每一项批评——奖励欺骗（第 2 课）、DPO（第 3 课）、谄媚（第 4 课）、CAI（第 5 课）、潜伏特工（第 7 课）、对齐伪装（第 9 课）——都在反驳这个流程的某个部分。奖励欺骗攻击第二阶段。DPO 合并了第二和第三阶段。CAI 替换了人类标注员。谄媚表明标注员是一个有偏信号。对齐伪装表明策略可以完全绕过第三阶段。如果不先在脑中建立这个流程，你无法理解任何这些批评。

## 使用它

`code/main.py` 在玩具偏好数据上模拟三个阶段。基础「策略」是动作 {A, B, C} 上的一个有偏硬币。第一阶段 SFT 在 200 个提示上模仿标注员动作。第二阶段在 500 个成对排序上拟合 Bradley-Terry 奖励模型。第三阶段运行一个简化的 PPO 更新，带有对 SFT 策略的 KL 惩罚。你可以观察奖励上升、KL 散度增长和策略漂移——你可以关闭 KL 项，看到奖励欺骗在 50 个更新步骤内出现。

要看什么：

- `beta = 0.1` vs `beta = 0.0` 的奖励轨迹。
- 训练步骤上的 KL(pi || pi_SFT)。
- 与标注员偏好相比的最终动作分布。

## 交付它

## 练习

1. 运行 `code/main.py`。设置 `beta = 0.0` 并报告 200 个 PPO 步骤后的动作分布。用一段话解释模式寻求行为。

2. 修改奖励模型，使其对动作 B 有 +0.5 的偏差（模拟的奖励 bug）。用 `beta = 0.1` 运行 PPO。KL 惩罚是否阻止策略利用偏差？在什么 `beta` 下利用变得可见？

3. 阅读 Ouyang 等人（arXiv:2203.02155）图 1。通过运行 1、5、20、100 步的 PPO 并测量相对于 SFT 模型的偏好来重现标注员偏好曲线。

4. 论文第 4.3 节报告 1.3B InstructGPT 在大约 70% 的情况下击败 175B GPT-3。为什么这个比例在隐藏生产提示上比在标注员自己的提示上更高？

5. 在相同的偏好数据上用 DPO（第 10 阶段 · 08）替换 PPO 损失。比较最终策略漂移（到 SFT 的 KL）和最终奖励。在匹配奖励下哪种方法漂移更远？

## 关键术语

| 术语 | 人们怎么说 | 它实际意味着什么 |
|------|-----------|-----------------|
| SFT | 「指令微调」 | 第一阶段：在提示-响应对上进行交叉熵微调 |
| 奖励模型 | 「RM」 | 在（提示，响应）上的标量回归器，用 Bradley-Terry 在成对标签上训练 |
| Bradley-Terry | 「成对偏好损失」 | -log sigmoid(r_w - r_l)；将成对排序归约为二分类 |
| KL 惩罚 | 「正则化器」 | `beta * KL(pi || pi_SFT)`——保持 RL 策略接近 SFT 锚点 |
| PPO-ptx | 「带预训练混合的 PPO」 | 将一部分预训练对数似然加入 PPO 目标以抵消对齐税 |
| 对齐税 | 「RLHF 退化」 | RLHF 后 RLHF 未针对的标准基准上的下降 |
| 标注员偏好 | 「真实标签」 | 人类排序的样本；RM 是它的统计代理，不是「人类价值观」的代理 |
